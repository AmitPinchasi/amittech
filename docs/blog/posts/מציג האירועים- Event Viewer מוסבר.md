# מציג האירועים - Event Viewer מוסבר

משהו קרס בווינדוס, המחשב התאתחל בלי סיבה נראית לעין, או שהתקנה נכשלה בלי הודעת שגיאה ברורה. במקום לנחש, יש כלי מובנה בווינדוס שמתעד כמעט כל אירוע משמעותי במערכת, ושמו מציג האירועים - Event Viewer. מי שיודע לקרוא אותו נכון פותר תקלות מהר משמעותית ממי שסתם מנחש.

## מה זה בעצם Event Viewer

Event Viewer הוא כלי מובנה בווינדוס שמציג את כל הלוגים שהמערכת, שירותי הרקע, והתוכנות המותקנות כותבים אליו. במקום שכל תוכנה תשמור את הלוגים שלה בפורמט משלה במקום אקראי, ווינדוס מספק מנגנון מרכזי ואחיד לתיעוד אירועים, וכלי גרפי אחד שיודע לקרוא את כולם. אפשר לפתוח אותו פשוט על ידי הקלדת "Event Viewer" בתפריט ההתחלה.

## שלושת היומנים המרכזיים

Event Viewer מחלק אירועים לכמה יומנים, ושלושה מהם הם הבסיס שכל משתמש כדאי שיכיר:

- **Application** - אירועים שקשורים לתוכנות שמותקנות במערכת. אם תוכנה קרסה, כאן תמצאו את השגיאה שהיא דיווחה עליה.
- **System** - אירועים שקשורים למערכת ההפעלה עצמה - דרייברים, שירותי ווינדוס, תהליכי אתחול וכיבוי. זה המקום הראשון לחפש בו כשמחשב מתאתחל מחדש בלי אזהרה.
- **Security** - אירועי אבטחה, כמו התחברויות מוצלחות וכושלות, שינויי הרשאות, וגישה לקבצים מבוקרים. זה היומן שרלוונטי במיוחד כשחוקרים חשד לפריצה או פעילות חשודה.

## רמות חומרה - להבין מה חשוב

לכל אירוע מוקצית רמת חומרה, שעוזרת לזהות מהר מה דורש תשומת לב:

- **מידע - Information** - אירוע רגיל, לרוב לא דורש שום פעולה, כמו שירות שהתחיל בהצלחה.
- **אזהרה - Warning** - משהו שכדאי לשים לב אליו, אבל לא בהכרח בעיה חמורה, כמו דיסק שמתקרב למגבלת שטח.
- **שגיאה - Error** - משהו שנכשל בפועל, ודורש בדרך כלל בדיקה.
- **קריטי - Critical** - כשל חמור שמשפיע ישירות על תפקוד המערכת.

בפועל, כשמחפשים סיבה לתקלה, הכי שימושי לסנן ולהציג רק אירועי Error ו-Critical, כי אירועי Information נוטים להיות המוני ולהציף את התמונה.

## איך בפועל מוצאים את מה שאתם מחפשים

הדרך היעילה ביותר לחפש תקלה היא לסנן לפי זמן - ברגע שאתם יודעים מתי בערך התקלה קרתה, אפשר לסנן את היומן להציג רק אירועים מהחלון הזמן הזה, מה שמצמצם מאות אירועים לכמה בודדים רלוונטיים. לכל אירוע יש גם קוד מזהה - Event ID - שאפשר לחפש באינטרנט כדי להבין בדיוק מה הוא אומר, כי הטקסט המוצג לפעמים טכני ולא ברור מיד.

בנוסף, Event Viewer מאפשר לבנות תצוגות מותאמות אישית, שמשלבות כמה קריטריונים - למשל רק אירועי Error מתוך יומן System, מתוזמן ב-24 השעות האחרונות - ולשמור אותן לשימוש חוזר.

## למה זה חשוב גם למי שלא מנהל מערכות

גם משתמש ביתי רגיל יכול להיעזר ב-Event Viewer כשמשהו לא מתנהג כרגיל - מחשב שמתאתחל מעצמו, תוכנה שקורסת בעקביות, או חשד שמישהו ניגש למחשב בלי רשות. במקרים כאלה, כמה דקות בEvent Viewer יכולות לתת תשובה ברורה במקום השערות.
