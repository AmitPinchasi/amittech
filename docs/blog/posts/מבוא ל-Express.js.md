# מבוא ל-Express.js

Node.js נותן לכם את היכולת הבסיסית להריץ קוד JavaScript בצד שרת, כולל מודול מובנה לטיפול בבקשות HTTP. אבל לכתוב שרת אמיתי עם המודול הגולמי הזה דורש הרבה קוד חוזר - פענוח כתובות, טיפול בפרמטרים, קריאת גוף בקשה, ניתוב לפי נתיב. Express.js נבנה בדיוק כדי לחסוך את כל זה, והוא הפך לפריימוורק הכי נפוץ לבניית API ב-Node.js.

## אז מה זה בעצם Express

Express הוא ספרייה קלת משקל שיושבת מעל Node.js, ונותנת שכבת נוחות לבניית שרתי אינטרנט ו-API. במקום לכתוב תנאים ידניים שבודקים איזו כתובת ביקשו ואיזו שיטת HTTP נשלחה, Express נותן ממשק פשוט - מגדירים נתיב, ומצמידים אליו פונקציה שמטפלת בו.

```
app.get('/users/:id', (req, res) => {
  res.json({ id: req.params.id });
});
```

שורה כזאת אומרת בדיוק מה שהיא נראית - כשמגיעה בקשת GET לנתיב עם מזהה משתמש, להריץ את הפונקציה הזאת ולהחזיר תשובה בפורמט JSON. אין צורך לפענח את הכתובת ידנית, Express עושה את זה מאחורי הקלעים.

## מנגנון ה-middleware - הרעיון המרכזי של Express

מה שבאמת הופך את Express לכלי עוצמתי הוא מנגנון ה-middleware. middleware הוא פונקציה שרצה על כל בקשה, או על חלק מהבקשות, לפני שהיא מגיעה ל-handler הסופי שלה. אפשר לחבר כמה middleware ברצף, וכל אחד מהם יכול לבדוק את הבקשה, לשנות אותה, או לעצור אותה לגמרי אם משהו לא תקין.

הדפוס הזה מאפשר להפריד לוגיקה שחוזרת בכל בקשה - בדיקת אימות, רישום ללוג, הגבלת קצב בקשות, פענוח גוף הבקשה - מהלוגיקה הספציפית של כל endpoint. במקום לכתוב את הבדיקה "האם המשתמש מחובר" בתוך כל handler בנפרד, כותבים middleware אחד שבודק את זה, ומצמידים אותו לכל הנתיבים שדורשים אימות. זה קוד נקי יותר, וגם קל יותר לתחזוקה - שינוי בלוגיקת האימות קורה במקום אחד בלבד.

## ניתוב מסודר לפרויקטים גדולים

בפרויקט קטן, כל הנתיבים יכולים לשבת בקובץ אחד. אבל ככל שה-API גדל, Express מאפשר לפצל נתיבים לקבצים נפרדים דרך מנגנון בשם Router - כל קובץ אחראי על קבוצת נתיבים הקשורים זה לזה, למשל כל מה שקשור למשתמשים בקובץ אחד, וכל מה שקשור להזמנות בקובץ אחר. זה הופך פרויקט עם מאות endpoints לניתן לניווט וללימוד, במקום קובץ ענק אחד שאף אחד לא רוצה לגעת בו.

## Express מול פריימוורקים אחרים

חלק מהמפתחים ששמים לב לפייתון מכירים את FastAPI כפריימוורק המקביל שם, שנותן גם ולידציה אוטומטית וגם תיעוד API אוטומטי. Express, לעומת זאת, נשאר מכוון, מינימליסטי בכוונה - הוא לא כופה עליכם מבנה או כלים נוספים, ומשאיר לכם לבחור בעצמכם ספריות לולידציה, לגישה למסד נתונים, ולכל דבר אחר. זה נותן גמישות מקסימלית, אבל גם דורש יותר החלטות בתחילת הפרויקט.

## למה זה הפריימוורק הראשון שכדאי ללמוד בNode.js

Express פשוט מספיק כדי להבין תוך שעה איך הוא עובד, אבל עשיר מספיק כדי לתמוך בפרויקטים אמיתיים בייצור. האקוסיסטם סביבו ענק - כמעט לכל צורך יש middleware מוכן שכתבו כבר אחרים, מה שחוסך זמן פיתוח משמעותי. זו הסיבה שכמעט כל מדריך ללימוד Node.js לצד שרת מתחיל דווקא ממנו.

## לסיכום

Express.js לקח את המודול הגולמי של Node.js והוסיף לו בדיוק את השכבה החסרה - ניתוב נוח ומנגנון middleware שמאפשר להפריד לוגיקה חוזרת מהלוגיקה הספציפית של כל endpoint. זו הסיבה שהוא נשאר, אחרי שנים רבות, אחת הבחירות הראשונות של כל מי שבונה API ב-Node.js.
