# ללמוד HTML ו-CSS מאפס - מדריך מעשי

HTML ו-CSS הן נקודת הכניסה הכי טבעית לעולם הפיתוח - התוצאה נראית לכם על המסך תוך דקות, בלי צורך בהתקנות מסובכות. הן גם בדיוק מה שהקורס שלנו מלמד מההתחלה. הנה מדריך מעשי איך בפועל מתחילים ללמוד אותן מאפס - לא "מה זה HTML", אלא מה כותבים ביום הראשון ואיך מתקדמים משם.

## מה צריך כדי להתחיל

שום דבר מיוחד. דפדפן וכל עורך טקסט - החל מ-Notepad ועד VS Code, שהוא הבחירה הכי נפוצה כי יש לו הדגשת תחביר ותצוגה מקדימה נוחה. אין קומפילציה, אין שרת, אין התקנות. שומרים קובץ עם סיומת html., פותחים אותו בדפדפן, ורואים תוצאה מיידית. זה חלק ממה שהופך את השילוב הזה לנקודת התחלה כל כך טובה למתחילים.

## תוכנית שבוע ראשון

- **ימים 1-2**: מבנה HTML בסיסי - התגיות html, head ו-body, כותרות (h1 עד h6), פסקאות (p), קישורים (a) ותמונות (img). תבנו דף "אודות" אישי פשוט.
- **ימים 3-4**: רשימות (ul, ol), טבלאות בסיסיות, וטפסים (form) עם שדות קלט. כאן מתחילים להבין איך אתר "מדבר" עם משתמש.
- **ימים 5-7**: CSS בסיסי - איך מחברים קובץ CSS ל-HTML, בוררים (selectors), צבעים, גופנים, ומרווחים (margin ו-padding). תתחילו לעצב את הדף שבניתם בימים הקודמים.

## הרעיון שחייבים להבין - box model

לפני שממשיכים הלאה ב-CSS, חייבים להבין את מודל הקופסה (box model) - כל אלמנט ב-HTML הוא בעצם קופסה מלבנית עם תוכן, padding (מרווח פנימי), border (מסגרת) ו-margin (מרווח חיצוני). רוב הבלבול של מתחילים ב-CSS - "למה יש רווח שאני לא מבין מאיפה", "למה שני אלמנטים לא נדבקים" - נובע מאי הבנה של המודל הזה. תשקיעו בו זמן לפני שאתם ממשיכים לפריסות (layout) מורכבות יותר.

## פלקסבוקס וגריד - איך פורסים דף אמיתי

אחרי היסודות, השלב הבא הוא ללמוד Flexbox ו-CSS Grid - שני כלים שפתרו בעיה שהטרידה מפתחים במשך שנים: איך מסדרים אלמנטים בצורה גמישה על המסך בלי טריקים מוזרים. Flexbox מצוין לסידור בשורה או עמודה אחת (כמו תפריט ניווט), ו-Grid מצוין לפריסות דו-ממדיות מורכבות יותר (כמו דף כתבה עם עמודות). אין צורך ללמוד את שניהם עד לפרטים הכי עמוקים בשבועות הראשונים - מספיק להבין מתי משתמשים בכל אחד.

## הפרויקט הראשון שכדאי לבנות

דף נחיתה (landing page) פשוט לעסק דמיוני - כותרת עליונה עם לוגו וניווט, אזור hero עם כותרת גדולה וכפתור קריאה לפעולה, אזור עם כמה שירותים או מוצרים בגריד, וטופס יצירת קשר בתחתית. זה פרויקט קלאסי כי הוא מכריח אתכם להשתמש כמעט בכל מה שלמדתם - מבנה סמנטי, פלקסבוקס או גריד, וגם עיצוב responsive בסיסי.

## נגישות וסמנטיקה - לא תוספת, יסוד

טעות נפוצה של מתחילים היא לבנות הכל מתגיות div ו-span בלבד. HTML מודרני כולל תגיות סמנטיות כמו header, main, nav, article ו-footer שנותנות משמעות אמיתית למבנה הדף - זה משפר גם נגישות למשתמשים עם קוראי מסך וגם דירוג בגוגל. תרגילו את עצמכם להשתמש בתגית הנכונה מההתחלה במקום ב-div לכל דבר.

## הטעויות הכי נפוצות של מתחילים

- **לנסות למרכז דבר עם מרווחים ידניים במקום עם Flexbox**. אם אתם משחקים עם margin כדי למרכז אלמנט, כנראה יש דרך נכונה יותר עם display: flex ו-justify-content.
- **לשכוח viewport meta tag**, מה שגורם לדף להיראות שבור בטלפון. שורה אחת בראש הדף פותרת את זה.
- **לערבב עיצוב בתוך התגית עם style="" בכל מקום** במקום קובץ CSS מסודר. זה עובד בהתחלה אבל הופך לבלגן תוך זמן קצר.
- **לא לבדוק את הדף ברוחבי מסך שונים**. עיצוב responsive זה לא תוספת בסוף - זה משהו שכדאי לבדוק כל הזמן תוך כדי בנייה.
