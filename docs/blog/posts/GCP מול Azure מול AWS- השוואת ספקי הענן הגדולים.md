# GCP מול Azure מול AWS- השוואת ספקי הענן הגדולים

מתישהו בדרך שלכם בהייטק תשמעו שיחה שמתחילה ב"אצלנו זה על AWS" או "אנחנו על Azure", ותרגישו קצת אבודים. שלושת הענקיות - AWS, Azure ו-GCP - מריצות היום את רוב האינטרנט, וכל אחת מהן טוענת שהיא הכי טובה. בואו נעשה סדר, בלי שיווק.

## AWS - השחקן הכי גדול ומורכב

Amazon Web Services היא ספקית הענן הראשונה שקמה ברצינות, וזה נותן לה יתרון עצום - היא פשוט הכי גדולה, עם הכי הרבה שירותים, הכי הרבה תיעוד, והכי הרבה אנשים שכבר עובדים איתה ויכולים לעזור כשנתקעים. החיסרון של הגודל הזה הוא מורכבות - יש ל-AWS מאות שירותים, וקל להרגיש מוצפים כשכל שירות נראה דומה לשירות אחר עם שם שונה מעט.

AWS נפוצה מאוד בסטארטאפים ובחברות טכנולוגיה, וזו הסיבה שרוב הקורסים והחומרים בעברית מתמקדים בה - היא פשוט הכי סביר שתפגשו אותה במקום העבודה הראשון שלכם.

## Azure - הבחירה של ארגונים גדולים

Azure, של מיקרוסופט, חזקה במיוחד בחברות גדולות וארגונים ותיקים, בעיקר כאלה שכבר משתמשים במוצרי מיקרוסופט אחרים כמו Windows Server, Active Directory, או Office 365. האינטגרציה הטבעית בין השירותים האלה הופכת את Azure לבחירה הגיונית לארגונים שכבר "חיים" בעולם של מיקרוסופט, ולכן אם אתם מתעניינים בניהול מערכות ווינדוס ארגוניות, סביר שתפגשו את Azure יותר מהאחרות.

## GCP - החוזק בנתונים ולמידת מכונה

Google Cloud Platform קטנה יותר בנתח שוק מהשתיים האחרות, אבל היא נהנית ממוניטין חזק בתחומי נתונים גדולים ולמידת מכונה - לא מפתיע, בהתחשב בכך שגוגל בעצמה מפעילה חלק מהמערכות המורכבות ביותר בעולם על התשתית הזאת. GCP גם נחשבת לרוב הידידותית ביותר מבחינת ממשק וחוויית משתמש, מה שהופך אותה לפופולרית בקרב חלק מהמתכנתים כבחירה אישית לפרויקטים קטנים.

## אז מה בעצם ההבדל המעשי

מתחת לשיווק, שלושתן עושות בערך את אותו דבר - נותנות לכם שרתים וירטואליים, אחסון קבצים, מסדי נתונים מנוהלים, ופונקציות serverless, רק תחת שמות שונים. שרת וירטואלי נקרא EC2 ב-AWS, Virtual Machines ב-Azure, ו-Compute Engine ב-GCP. אחסון אובייקטים נקרא S3, Blob Storage, ו-Cloud Storage בהתאמה. ברגע שמבינים את הרעיון הבסיסי באחת מהן, המעבר לשנייה הוא בעיקר עניין של לימוד שמות חדשים לאותם מושגים.

## אז איזו לבחור כמתחיל

התשובה הכנה - כמעט לא משנה, וזאת לא תשובה מתחמקת. מה שבאמת חשוב זה להבין את המושגים הבסיסיים של ענן - מה זה שרת וירטואלי, מה זה אחסון אובייקטים, איך מנהלים הרשאות בין שירותים, ואיך חושבים על עלויות. ברגע שיש לכם את הבסיס הזה מספק אחד, מעבר לספק אחר, כולל בעבודה עצמה כשמעביר אתכם ממקום שעובד על AWS למקום שעובד על GCP, לוקח ימים ולא חודשים.

בכל זאת, אם אתם רק מתחילים ורוצים חומר לימוד בעברית וזמין, AWS היא ההמלצה המעשית - יש לה הכי הרבה נוכחות בשוק הישראלי, והכי הרבה משרות שדורשות ניסיון איתה ספציפית.
