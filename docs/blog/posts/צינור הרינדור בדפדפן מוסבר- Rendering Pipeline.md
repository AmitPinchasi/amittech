# צינור הרינדור בדפדפן מוסבר - Rendering Pipeline

כשקוד HTML ו-CSS מגיעים לדפדפן, קורה תהליך שלם עד שהם הופכים לפיקסלים צבעוניים על המסך. רוב המפתחים מעולם לא חושבים על התהליך הזה, אבל הבנה בסיסית שלו היא ההבדל בין קוד שמרגיש "כבד" לקוד שרץ חלק גם באנימציות מורכבות.

## חמשת השלבים העיקריים

- **ניתוח - Parsing.** הדפדפן קורא את קובץ ה-HTML ובונה ממנו עץ אובייקטים שנקרא DOM. במקביל, הוא קורא את ה-CSS ובונה עץ מקביל שנקרא CSSOM, שמייצג את כל כללי הסגנון.
- **בניית עץ הרינדור - Render Tree.** הדפדפן משלב את ה-DOM וה-CSSOM לעץ אחד שמכיל רק את מה שבאמת יוצג על המסך (אלמנטים עם display: none, למשל, לא נכנסים לעץ הזה).
- **פריסה - Layout.** הדפדפן מחשב את המיקום המדויק והגודל של כל אלמנט בעץ, בהתאם לגודל המסך, המרווחים, וכל שאר הכללים.
- **צביעה - Paint.** הדפדפן ממלא כל אלמנט בפועל - צבעים, טקסט, גבולות, צללים.
- **הרכבה - Composite.** הדפדפן מרכיב את כל השכבות (layers) שנצבעו יחד לתמונה סופית אחת שמוצגת על המסך.

## למה זה חשוב לביצועים

הבעיה מתחילה כשמשהו משתנה בדף אחרי הטעינה הראשונית - לדוגמה, אנימציה, גלילה, או פעולת משתמש. הדפדפן צריך להריץ מחדש חלק מהתהליך הזה, ופה נכנסים שני מושגים קריטיים:

- **Reflow (או Layout מחדש).** קורה כשמשהו שמשפיע על מיקום או גודל של אלמנטים משתנה, למשל שינוי ב-width או הוספת אלמנט חדש. זה מכריח את הדפדפן לחשב מחדש את כל הפריסה, ולפעמים אפילו לגרור אחריו reflow של אלמנטים אחרים בדף.
- **Repaint.** קורה כשמשהו חזותי משתנה בלי להשפיע על מיקום, כמו צבע רקע. זול יותר מ-reflow, אבל עדיין לא חינם.

## למה transform ו-opacity הם "זולים"

יש קבוצה קטנה של תכונות CSS שהדפדפן יכול לשנות בלי לעבור דרך שלבי layout ו-paint בכלל, ולטפל בהן ישירות בשלב ה-composite. transform ו-opacity הן הדוגמאות המרכזיות. זו הסיבה שהמלצה חוזרת בכל מקום שעוסק באנימציות היא להשתמש ב-transform: translateX() במקום לשנות left, ובאנימציית opacity במקום להחליף display.

ההבדל בפועל הוא שאנימציה עם transform יכולה לרוץ חלק גם על מכשירים חלשים, כי הדפדפן פשוט מזיז שכבה מוכנה, בלי לחשב מחדש שום דבר על שאר הדף.

## דוגמה למה שכדאי להימנע ממנו

```css
/* יקר - גורם ל-reflow בכל frame */
.box {
  transition: left 0.3s;
}

/* זול - רק שכבת composite */
.box {
  transition: transform 0.3s;
}
```

השינוי הזה נראה קטן, אבל בדף עם הרבה אלמנטים הוא יכול להיות ההבדל בין אנימציה חלקה לאנימציה מקוטעת (jank).

## מה עושים עם הידע הזה בפועל

לא צריך לשנן את כל שלבי ה-pipeline בעל פה, אבל שווה לזכור שני כללים: להעדיף transform ו-opacity לאנימציות, ולהימנע משינויים תכופים בתכונות layout כמו width, height או top כשמדובר בקוד שרץ הרבה פעמים בשנייה, למשל בזמן גלילה.
