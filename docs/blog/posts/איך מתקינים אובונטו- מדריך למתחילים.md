# איך מתקינים אובונטו - מדריך למתחילים

אובונטו היא ההפצה הכי מומלצת למי שמתחיל את המסע עם לינוקס, בזכות קהילה ענקית, תיעוד מצוין, ותהליך התקנה שהתייעל מאוד לאורך השנים. הנה מדריך מעשי שיעביר אתכם מקובץ ISO ריק למערכת עובדת.

## שלב ראשון - בוחרים איך תתקינו

לפני שמורידים כלום, כדאי להחליט איך אתם רוצים להריץ את אובונטו:

- **במכונה וירטואלית** - הדרך הכי בטוחה למתחילים. אתם מתקינים אובונטו בתוך תוכנה כמו VirtualBox, בלי לגעת בכלל במחשב הפיזי שלכם.
- **בהתקנה כפולה - dual boot** - אובונטו מותקן לצד ווינדוס הקיים, ובכל הפעלה בוחרים איזו מערכת להריץ. מתקדם יותר, ודורש זהירות עם חלוקת הדיסק.
- **כמערכת יחידה** - מוחקים את מה שקיים ומתקינים רק אובונטו. מתאים למי שכבר בטוח בבחירה.

למתחילים, מומלץ בחום להתחיל במכונה וירטואלית.

## שלב שני - מורידים את קובץ ה-ISO

נכנסים לאתר הרשמי של אובונטו ומורידים את הגרסה האחרונה, שנקראת גרסת LTS - Long Term Support. גרסאות LTS מקבלות תמיכה ועדכוני אבטחה לתקופה ארוכה בהרבה מגרסאות הביניים, ולכן הן הבחירה הבטוחה למי שמתחיל.

הקובץ שתורידו הוא קובץ ISO - תמונת דיסק שלמה, שממנה אפשר להתקין את המערכת.

## שלב שלישי - יצירת מדיה להתקנה

אם אתם מתקינים במכונה וירטואלית, אין צורך ביותר מזה - פשוט מצביעים את המכונה הווירטואלית על קובץ ה-ISO. אם אתם מתקינים ישירות על מחשב פיזי, תצטרכו ליצור USB בר-אתחול, באמצעות כלי כמו Rufus או balenaEtcher, שמעתיקים את תוכן ה-ISO לזיכרון נייד בצורה שהמחשב יודע לאתחל ממנה.

## שלב רביעי - מריצים את ההתקנה

מפעילים את המכונה הווירטואלית, או מאתחלים את המחשב מה-USB, ועוברים את אשף ההתקנה של אובונטו:

1. בוחרים שפה ופריסת מקלדת.
2. מחליטים אם להתקין תוכנות נוספות מראש, כמו נגני מדיה וקודקים.
3. בוחרים את סוג ההתקנה - אם זו מכונה וירטואלית או מחשב ריק, האפשרות הפשוטה ביותר היא "מחק דיסק והתקן אובונטו".
4. יוצרים משתמש ראשי, בוחרים שם מחשב וסיסמה.
5. ממתינים שההתקנה תסתיים ומאתחלים מחדש.

בתוך פחות מ-20 דקות ברוב המקרים, יש לכם מערכת אובונטו מוכנה לעבודה.

## אחרי ההתקנה - כמה דברים ראשונים

ברגע שהמערכת עולה, כדאי לעדכן אותה מיד לגרסאות התוכנה העדכניות ביותר, דרך אפליקציית "עדכוני תוכנה" או בטרמינל עם `sudo apt update && sudo apt upgrade`. אם התקנתם במכונה וירטואלית, כדאי גם להתקין את חבילת ה-Guest Additions או Guest Tools המתאימה, שמשפרת ביצועים ומאפשרת מסך מלא נוח יותר.

## טעויות נפוצות שכדאי להימנע מהן

הטעות הכי נפוצה היא לתת למכונה הווירטואלית פחות מדי משאבים - נסו להקצות לפחות 2 גיגה זיכרון ו-20 גיגה שטח דיסק, אחרת המערכת תרגיש איטית מאוד. טעות נוספת היא לוותר על סיסמה, מה שעלול ליצור בעיות הרשאה מאוחר יותר כשתצטרכו הרשאות מנהל - sudo.
