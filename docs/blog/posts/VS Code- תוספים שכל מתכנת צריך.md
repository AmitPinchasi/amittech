# VS Code - תוספים שכל מתכנת צריך

VS Code בהתקנה נקייה הוא כבר עורך קוד מצוין, אבל הכוח האמיתי שלו מגיע מהתוספים - Extensions. הבעיה היא שבחנות התוספים יש אלפי אפשרויות, וקל מאוד ליפול למלכודת של להתקין עשרים תוספים ביום הראשון ולא באמת לדעת מה כל אחד עושה. בואו נעבור על מה שבאמת שווה את זה.

## GitLens - לראות את ההיסטוריה בלי לצאת מהקוד

אם אתם עובדים עם git, וכל מתכנת צריך, GitLens הוא כנראה התוסף הכי שימושי שתתקינו. הוא מציג לצד כל שורת קוד מי כתב אותה, מתי, ובאיזה commit, ישירות בתוך הקובץ. במקום לקפוץ לטרמינל כל פעם שאתם רוצים להבין למה שורה מסוימת נראית ככה, יש לכם את התשובה מיד, בלי לעזוב את העורך.

## Python ו-Pylance - חובה למי שעובד עם פייתון

תוסף Python של מיקרוסופט נותן לכם הדגשת תחביר נכונה, הרצה ודיבוג ישירות מתוך העורך, וזיהוי אוטומטי של סביבות וירטואליות. Pylance משלים אותו עם auto-complete חכם, זיהוי שגיאות סוג לפני שהקוד בכלל רץ, וניווט מהיר בין פונקציות. שני התוספים האלה יחד הופכים כתיבת פייתון ב-VS Code מנוחה לנוחה משמעותית יותר.

## Error Lens - שגיאות שאי אפשר לפספס

בברירת מחדל, VS Code מסמן שגיאות בקו אדום דק מתחת לקוד, שקל לפספס אם לא מסתכלים ישר עליו. Error Lens לוקח את אותה שגיאה ומציג אותה כטקסט מלא ובולט ישר לצד השורה. זה שינוי קטן שחוסך המון זמן, במיוחד למתחילים שעוד לומדים לזהות שגיאות נפוצות.

## Code Spell Checker - הצלה משגיאות כתיב מביכות

תוסף פשוט שבודק איות באנגלית בקוד, בהערות, ובמשתני שם. נשמע שולי, אבל שם משתנה עם שגיאת כתיב, או תיעוד עם שגיאה, נראה לא מקצועי כשמישהו אחר סוקר את הקוד שלכם - ובמיוחד ב-Pull Request פומבי.

## Prettier או Black - עיצוב אוטומטי של קוד

במקום לדאוג ידנית לרווחים, אינדנטציה, ומרכאות עקביות, תוסף פורמט אוטומטי מסדר את זה בשבילכם בלחיצת שמירה. Prettier מתאים למגוון רחב של שפות כולל JavaScript, ו-Black מיועד ספציפית לפייתון. מרגע שהתרגלתם לזה, פשוט אי אפשר לחזור לעיצוב ידני.

## Live Share - לתכנת ביחד מרחוק

תוסף שנותן לשני אנשים לעבוד על אותו קובץ, באותו זמן, כל אחד מהמחשב שלו, ממש כמו מסמך גוגל משותף אבל לקוד. שימושי מאוד ללמידה משותפת, לעזרה למי שנתקע, או לזוגות תכנות מרחוק.

## Todo Tree - למצוא את כל ה"אני אחזור לזה אחר כך"

מתכנתים משאירים בקוד הערות כמו `# TODO: לתקן את זה` כל הזמן, ואז שוכחים מהן. Todo Tree סורק את כל הפרויקט ומרכז את כל ההערות האלה ברשימה אחת נגישה, כדי שהן לא ייעלמו בין אלפי שורות קוד.

## REST Client - לבדוק API בלי לעזוב את העורך

לחלופין ל-Postman כשעובדים עם API, REST Client מאפשר לשלוח בקשות ולראות תשובות ישירות בתוך VS Code, בקובץ טקסט פשוט. נוח במיוחד כי אפשר לשמור את הבקשות בתוך הפרויקט עצמו ולשתף אותן עם אחרים.
