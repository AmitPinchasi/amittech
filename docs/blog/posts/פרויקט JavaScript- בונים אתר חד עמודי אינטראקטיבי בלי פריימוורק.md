# האתר החד עמודי שילמד אתכם JavaScript אמיתי, בלי קיצורי דרך

יש פיתוי גדול אצל כל מי שמתחיל ללמוד פיתוח אתרים - לקפוץ ישר לפריימוורק, כי "ככה עובדים בתעשייה". זה נכון בטווח הארוך, אבל זו טעות בטווח הקצר. אתר חד עמודי אינטראקטיבי שבנוי ב-JavaScript טהור, בלי React ובלי שום ספריה, הוא אחד הפרויקטים הכי משתלמים שאפשר לבנות, כי הוא חושף אתכם ישירות ל-API-ים שכל פריימוורק בעולם בסופו של דבר עוטף.

## למה בכלל ללמוד DOM טהור

כשאתם כותבים document.querySelector וaddEventListener בעצמכם, אתם לומדים בדיוק מה קורה מתחת למכסה המנוע כשפריימוורק "מטפל" בזה בשבילכם. זו לא רומנטיקה של "פעם זה היה קשה יותר" - זו הבנה מעשית שעוזרת לכם לדבג בעיות, גם כשאתם עובדים בתוך פריימוורק. מפתח שמבין DOM טהור פותר בעיה בפריימוורק תוך דקות. מפתח שלא, נתקע ומחפש בגוגל בלי להבין מה הוא בכלל מחפש.

## מתכננים את הדפים בתוך העמוד האחד

אתר חד עמודי (single page) הוא בעצם קובץ HTML אחד עם כמה אזורים - נניח "בית", "אודות", "שירותים" ו"צור קשר" - שכולם קיימים כל הזמן בדף, ותפריט הניווט פשוט גולל אליהם. תכננו מראש אילו אזורים יהיו, ואיך הם מסודרים, לפני שאתם נוגעים בקוד. זה מונע מכם למצוא את עצמכם משנים את מבנה ה-HTML כל חמש דקות.

## גלילה חלקה בין אזורים

הפיצ'ר הכי בולט בכל אתר חד עמודי הוא לחיצה על קישור בתפריט שגורמת לגלילה חלקה, לא קפיצה מיידית. הפתרון המודרני פשוט מאוד - שימוש ב-scrollIntoView עם אפשרות behavior: smooth, בלי צורך בשום ספריית אנימציה חיצונית.

```javascript
document.querySelectorAll('nav a').forEach(link => {
  link.addEventListener('click', (e) => {
    e.preventDefault();
    const target = document.querySelector(link.getAttribute('href'));
    target.scrollIntoView({ behavior: 'smooth' });
  });
});
```

## event delegation - למה לא לצרף מאזין לכל אלמנט

אם יש לכם עשרות קישורים או כפתורים, צירוף מאזין אירועים (event listener) לכל אחד בנפרד עובד, אבל זה לא מדויק. הדרך הנכונה, שנקראת event delegation, היא לצרף מאזין אחד לאלמנט הורה משותף, ולבדוק בתוך הפונקציה איזה אלמנט ילד בדיוק נלחץ, דרך event.target. זה חשוב במיוחד כשיש תוכן שנוצר דינמית אחרי טעינת הדף, כי מאזין שצורף ישירות לאלמנט שלא היה קיים בזמן טעינת הדף פשוט לא יעבוד.

```javascript
document.querySelector('.menu').addEventListener('click', (e) => {
  if (e.target.matches('a')) {
    handleNavClick(e.target);
  }
});
```

## ולידציה לטפסים בלי ספריה

טופס יצירת קשר הוא הזדמנות מעולה לתרגל ולידציה - בדיקה שהאימייל בפורמט תקין, שהשדות החובה לא ריקים, והצגת הודעת שגיאה ברורה ליד השדה הבעייתי, לא רק "משהו השתבש" גנרי למעלה. ה-API הטבעי לכך הוא checkValidity ו-setCustomValidity, אבל גם ולידציה ידנית עם ביטויים רגולריים פשוטים עובדת מצוין ומלמדת יותר.

## אנימציות פשוטות עם CSS ולא עם JavaScript

טעות נפוצה היא לנסות לכתוב אנימציות (fade-in, שינויי צבע) לגמרי ב-JavaScript. ברוב המקרים עדיף להוסיף ולהסיר class עם JavaScript, ולתת ל-CSS לטפל באנימציה עצמה דרך transition. זה גם ביצועים טובים יותר, וגם קוד JavaScript נקי הרבה יותר לקריאה. הרעיון של איך לחלק אחריות בין HTML, CSS ו-JavaScript הוא בדיוק מה שנלמד לעומק בקורס צד לקוח שלנו, עוד לפני שמגיעים לפריימוורקים.

## המלכודות הנפוצות

טעות ראשונה - לשכוח preventDefault בקישורי ניווט, מה שגורם לדפדפן לקפוץ במקום לגלול חלק. טעות שנייה - לכתוב את כל הקוד כגוש אחד בקובץ JavaScript ענק במקום לפצל לפונקציות קטנות עם שם ברור לכל אחת. טעות שלישית - להריץ קוד שמחפש אלמנטים בדף לפני שהדף בכלל נטען במלואו, מה שדורש לעטוף הכל ב-DOMContentLoaded.
