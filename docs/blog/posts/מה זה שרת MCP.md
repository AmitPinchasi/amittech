# מה זה שרת MCP - ההסבר הטכני הקצר על החלק שבאמת עושה את העבודה

בפוסט אחר הסברתי מה זה בכלל פרוטוקול MCP ברמת הרעיון - סוג של "שפה משותפת" בין מודלים לכלים. הפעם אני רוצה להתמקד בצד הקונקרטי - מה זה בעצם שרת MCP, איך הוא בנוי, ואיך זה עובד כשאתם מחברים אחד כזה ל-Claude Code.

## שרת MCP הוא פשוט תוכנית קטנה

שרת MCP הוא לא "שרת" במובן שהתרגלנו אליו - הוא לא בהכרח רץ על מכונה מרוחקת. זו תוכנית, לפעמים ממש קטנה, שיודעת "לדבר" בפרוטוקול MCP וחושפת יכולות מסוימות. הוא יכול לרוץ מקומית על המחשב שלכם, לצד Claude Code, או להיות מאוחסן מרחוק ולהתחבר אליו דרך הרשת. מה שמגדיר אותו הוא לא המיקום, אלא זה שהוא עונה לפי הפרוטוקול המוסכם.

כשמדברים על מה שרת MCP "חושף", בדרך כלל מתכוונים לשני סוגי דברים - כלים - tools - שהמודל יכול להפעיל, כמו "חפש בגיטהאב" או "כתוב לקובץ", ומשאבים - resources - שהמודל יכול לקרוא, כמו תוכן של מסמך או תוצאת שאילתה למסד נתונים.

## דוגמאות לשרתים נפוצים

יש כבר עולם שלם של שרתי MCP קיימים לשימוש. שרת לגיטהאב שמאפשר לסוכן קוד לקרוא issues, לפתוח pull requests, ולבדוק סטטוס בילדים. שרת למערכת קבצים שנותן גישה מבוקרת לתיקיות ספציפיות במחשב. שרתים למסדי נתונים שמאפשרים לשלוח שאילתות ולקבל תוצאות. שרתים לכלי תקשורת כמו סלאק, שמאפשרים לסוכן לקרוא ולשלוח הודעות. חלק גדול מהערך של האקוסיסטם הזה הוא שלא צריך לבנות את השרתים האלה בעצמכם - הרבה מהם כבר קיימים וזמינים לחיבור.

## איך זה מתחבר בפועל ל-Claude Code

כשמגדירים שרת MCP בכלי כמו Claude Code, בפועל אתם מוסיפים הגדרה שאומרת "הנה שרת נוסף, ואלה הכלים שאני מרשה לך להשתמש בהם ממנו". מהרגע הזה, כשאתם מתארים משימה שרלוונטית לאותו שרת - נניח "תבדוק אם יש issue פתוח שקשור לבאג הזה" - הסוכן יודע שיש לו כלי מתאים לזה, ומשתמש בו כחלק טבעי מהעבודה, בדיוק כמו שהוא משתמש בכלי לקריאת קבצים.

## למה זה כל כך שונה מלבנות אינטגרציה בעצמכם

לפני MCP, אם רציתם לחבר כלי AI לגיטהאב, הייתם צריכים לבנות קוד ייעודי שמתרגם בין הבקשות של המודל הספציפי שלכם לבין ה-API של גיטהאב. אם רציתם לחבר גם לסלאק, עוד קוד ייעודי, שוב מתחילים מאפס. שרת MCP נכתב פעם אחת לפי התקן, ואז כל לקוח MCP - כל כלי שתומך בפרוטוקול - יכול להשתמש בו בלי שינוי נוסף. זה בדיוק ההבדל בין לבנות כבל מותאם אישית לכל מכשיר, לבין תקן חיבור אחיד.

## שיקולי אבטחה שחשוב לא לדלג עליהם

כאן מגיע החלק שאני חושב שהכי חשוב להדגיש. כל שרת MCP שאתם מחברים הוא בעצם דלת שאתם פותחים - הוא נותן לסוכן AI יכולת לפעול בעולם האמיתי, לא רק לענות טקסט. לפני שמחברים שרת חדש, כדאי לשאול - איזה הרשאות בדיוק הוא דורש, האם הוא בא ממקור אמין, ומה הנזק הפוטנציאלי אם הסוכן ישתמש בו בצורה לא צפויה. שרת MCP שמחובר למסד נתונים בפרודקשן עם הרשאות כתיבה מלאות הוא סיפור אחר לגמרי משרת שקורא קבצים בתיקייה מקומית מבודדת. תמיד כדאי לתת את מינימום ההרשאות הדרוש למשימה.
