# פקודות רשת בלינוקס - ip, ss ו-netstat מוסברות

מנהל מערכות ותיק ומנהל מערכות חדש עשויים להשתמש בכלים שונים לגמרי כדי לבדוק בדיוק אותו דבר - מצב הרשת של השרת. זה קורה כי עולם כלי הרשת בלינוקס עבר שינוי משמעותי בשנים האחרונות, וכדאי להכיר גם את הישן וגם את החדש.

## ip - הכלי המרכזי של היום

כלי ip- הוא הכלי המודרני והמרכזי לניהול רשת בלינוקס היום, וכבר החליף כמעט לגמרי כלים ישנים יותר כמו ifconfig- ו-route-. הוא מאורגן סביב אובייקטים - כתובות, ממשקים, נתיבים - שעליהם אפשר לבצע פעולות.

להצגת כל ממשקי הרשת והכתובות שמוקצות להם:

```bash
ip addr show
```

הפקודה הזו מציגה כל כרטיס רשת במערכת, כולל כתובת ה-IP שמוקצית לו, אם בכלל. גרסה מקוצרת נפוצה היא ip a-.

להצגת טבלת הניתוב, שמראה לאן המערכת שולחת תעבורה יוצאת:

```bash
ip route show
```

זה שימושי מאוד כשמנסים להבין למה תעבורה יוצאת לא מגיעה ליעד הצפוי - לפעמים הבעיה היא נתיב ניתוב שגוי, לא בעיה ברשת עצמה.

## ss - התחליף המהיר ל-netstat

בעבר, netstat- הייתה הפקודה הסטנדרטית להצגת חיבורי רשת פעילים ופורטים פתוחים. היום היא נחשבת מיושנת, ובהרבה הפצות מודרניות היא אפילו לא מותקנת כברירת מחדל. התחליף שלה נקרא ss-, ראשי תיבות של socket statistics, והוא מהיר ומדויק יותר, במיוחד במערכות עם הרבה חיבורים פעילים.

```bash
ss -tuln
```

הדגלים כאן שימושיים במיוחד - t- מציג חיבורי TCP, u- מציג חיבורי UDP, l- מציג רק פורטים שמאזינים לחיבורים נכנסים, ו-n- מציג כתובות ופורטים כמספרים, בלי לנסות לתרגם אותם לשמות שגורם להצגה להיות איטית יותר.

```bash
ss -tp
```

הפקודה הזו מוסיפה את הדגל p-, שמציג גם איזה תהליך מחזיק בכל חיבור, שימושי מאוד כשמנסים למצוא איזו תוכנית תופסת פורט מסוים.

## netstat - עדיין שימושי לדעת

למרות ש-ss- הוא הכלי המומלץ היום, עדיין שווה להכיר את netstat-, כי הוא מופיע במערכות ישנות רבות, ובתסריטים ובמדריכים ותיקים שעדיין מסתובבים ברשת. הפקודה השקולה ל-ss -tuln- ב-netstat- היא:

```bash
netstat -tuln
```

מבחינת התוצאה, שתי הפקודות מציגות מידע דומה מאוד, אבל ss- עושה זאת מהר יותר, וזו הסיבה שהיא הפכה לברירת המחדל.

## שימושים נפוצים באבחון תקלות

השילוב של הכלים האלה שימושי מאוד באבחון בעיות רשת נפוצות:

- **בדיקה שהשירות בכלל מאזין** - אם שרת אתרים אמור להאזין בפורט 80, ss -tuln- מראה מיד אם הוא באמת מאזין, ובאיזה ממשק - רק local host, או גם חיבורים חיצוניים.
- **מציאת תהליך שתופס פורט** - כשמנסים להפעיל שירות ומקבלים שגיאה שהפורט כבר בשימוש, ss -tp- מראה בדיוק איזה תהליך אחראי לזה.
- **בדיקת כתובת וניתוב** - ip addr- ו-ip route- עוזרים לוודא שהשרת עצמו מוגדר נכון ברשת, לפני שמחפשים את הבעיה במקום אחר.

## לסיכום
