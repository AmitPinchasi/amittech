# כלי אבחון רשת בווינדוס מוסברים

"אין לי אינטרנט" יכולה לנבוע מעשרות סיבות שונות - הראוטר, ספק האינטרנט, הגדרת רשת שגויה, או אפילו האתר הספציפי שניסיתם לגשת אליו. במקום לנחש, ווינדוס מגיע עם כמה כלי שורת פקודה ותיקים ופשוטים שמאפשרים לצמצם את הבעיה תוך דקה, בלי שום תוכנה נוספת.

## ipconfig - מה בכלל המצב שלי ברשת

הצעד הראשון בכל אבחון הוא לבדוק מה ההגדרות הנוכחיות של המחשב עצמו:

```
ipconfig /all
```

הפקודה הזו מציגה את כתובת ה-IP של המחשב, subnet mask, ה-gateway המוגדר (בדרך כלל הראוטר), ושרתי DNS. אם המחשב מציג כתובת שמתחילה ב-169.254, זה סימן ברור שהוא לא הצליח לקבל כתובת תקינה משרת DHCP, ושהבעיה כנראה בחיבור עצמו, עוד לפני שהגענו לאינטרנט.

פקודה נוספת שימושית היא:

```
ipconfig /flushdns
```

שמנקה את המטמון המקומי של תרגומי שמות דומיין, שימושי כשאתר לא נטען כי המחשב עדיין "זוכר" כתובת ישנה ולא נכונה שלו.

## ping - האם היעד בכלל מגיב

ping- היא כנראה הפקודה הכי ותיקה ובסיסית באבחון רשת. היא שולחת חבילה קטנה ליעד ומחכה לתשובה, ומודדת כמה זמן זה לקח:

```
ping google.com
```

אם ping ל-google.com מצליח, זה סימן שהחיבור לאינטרנט תקין ברמה הבסיסית. אם ping לא מצליח, שווה לבדוק אם ping לכתובת ה-gateway מהראוטר עצמו כן עובד - אם כן, הבעיה כנראה אצל ספק האינטרנט או הרחק יותר ברשת, לא בבית עצמו.

## tracert - איפה בדיוק בדרך זה נשבר

אם ping מראה שיש בעיה, tracert- (ראשי תיבות של trace route) עוזר להבין איפה בדיוק בדרך התעבורה נתקעת:

```
tracert google.com
```

הפקודה הזו מציגה כל קפיצה - hop - שהתעבורה עוברת בדרך ליעד, מהראוטר הביתי, דרך ספק האינטרנט, ועד היעד הסופי. אם התעבורה מגיעה בהצלחה עד קפיצה מסוימת ואז נתקעת, יש לכם אינדיקציה ברורה איפה בדיוק בשרשרת הבעיה נמצאת - לפעמים אצלכם, לפעמים אצל הספק, ולפעמים אפילו אצל היעד עצמו.

## nslookup - האם שם הדומיין בכלל מתורגם נכון

לפעמים הבעיה היא לא ברשת עצמה, אלא בתרגום שם הדומיין לכתובת IP. nslookup- בודק בדיוק את זה:

```
nslookup google.com
```

הפקודה מציגה איזו כתובת IP שרת ה-DNS מחזיר עבור השם שהוזן. אם הפקודה נכשלת או מחזירה כתובת מוזרה ולא צפויה, הבעיה כנראה קשורה לשרתי DNS - אולי הם לא זמינים, ואולי משהו במחשב שינה אותם בלי ידיעתכם.

## איך משלבים את כל הכלים יחד לאבחון מסודר

סדר אבחון הגיוני נראה כך - קודם ipconfig- לוודא שהמחשב עצמו קיבל כתובת תקינה. אחר כך ping- לgateway- כדי לוודא שהחיבור לראוטר תקין. אז ping- לכתובת IP חיצונית ידועה, כדי לבדוק חיבור בסיסי לאינטרנט בלי תלות ב-DNS. ולבסוף nslookup- ו-ping- לשם דומיין, כדי לוודא שגם תרגום השמות עובד. אם כל שלב עובר בהצלחה עד שלב מסוים ואז נכשל, יש לכם בדיוק את המקום לחפש בו את הבעיה.

## לסיכום

ארבעת הכלים האלה, ipconfig, ping, tracert ו-nslookup, קיימים בכל מחשב ווינדוס ללא יוצא מן הכלל, ומאפשרים אבחון שיטתי של כמעט כל בעיית רשת בסיסית, בלי שום התקנה נוספת. השקעה של כמה דקות בלמידת הכלים האלה חוסכת שעות של ניחושים בפעם הבאה שמשהו ברשת פשוט לא עובד.
