בהרצאה הקודמת בנינו לולאת כלים בעצמנו: המודל ביקש קריאה, אנחנו הרצנו, החזרנו תוצאה, וחזרנו על זה עד שהמודל סיים. זה עבד, אבל שמנו לב לכמה דברים - היינו צריכים לכתוב את הלולאה, לנהל את היסטוריית ההודעות, לממש כל כלי בעצמנו (קריאת קבצים, הרצת פקודות, חיפוש). מה אם כל זה כבר היה מוכן? זו בדיוק המטרה של **ה-Agent SDK** של Claude: הוא נותן לנו את אותו מנוע שמפעיל את Claude Code - אותם כלים מובנים, אותה לולאת סוכן, אותו ניהול הקשר - כספרייה שאפשר להפעיל מקוד פייתון או TypeScript.

בהרצאה הזו נבין מה ה-Agent SDK נותן מעל קריאות API רגילות, נכיר את הפונקציה המרכזית שלו, ונבנה סוכן מינימלי שקורא קבצים, מריץ פקודות ופותר משימה בעצמו.

---

## מהו ה-Agent SDK - the Agent SDK

חשוב להבחין בין שני דברים שנשמעים דומים:

- **ה-Client SDK** (`anthropic` / `@anthropic-ai/sdk`) - זה מה שהשתמשנו בו עד עכשיו. הוא נותן גישה ישירה ל-API. אנחנו כותבים את לולאת הכלים ומממשים כל כלי.
- **ה-Agent SDK** (`claude-agent-sdk` / `@anthropic-ai/claude-agent-sdk`) - זה Claude Code ארוז כספרייה. הוא מגיע עם כלים מובנים, עם לולאת הסוכן המלאה, עם ניהול הקשר, hooks, תת-סוכנים, הרשאות וסשנים. אנחנו רק קוראים ל-`query(prompt, options)` והוא מנהל הכל.

ההבדל במשפט אחד: עם ה-Client SDK **אתה** כותב את לולאת הכלים; עם ה-Agent SDK **Claude** מנהל אותה בעצמו, עם כלים שכבר קיימים.

```python
# Client SDK: אתה מממש את לולאת הכלים
response = client.messages.create(...)
while response.stop_reason == "tool_use":
    result = your_tool_executor(response.content)
    response = client.messages.create(...)

# Agent SDK: Claude מנהל את הכלים אוטונומית
async for message in query(prompt="תקן את הבאג ב-auth.py"):
    print(message)
```

## הסוכן הראשון - the first agent

הפונקציה המרכזית בפייתון היא `query`, ואת ההגדרות מעבירים דרך `ClaudeAgentOptions`. הנה סוכן שמסתכל על התיקייה ומדווח אילו קבצים יש בה:

```python
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions


async def main():
    async for message in query(
        prompt="אילו קבצים יש בתיקייה הזאת?",
        options=ClaudeAgentOptions(allowed_tools=["Bash", "Glob"]),
    ):
        if hasattr(message, "result"):
            print(message.result)


asyncio.run(main())
```

מה קרה כאן? נתנו ל-`query` הנחיה במחרוזת, ורשימת הכלים שהסוכן רשאי להשתמש בהם דרך `allowed_tools`. הסוכן קיבל את המשימה, החליט בעצמו להשתמש ב-`Glob` וב-`Bash`, סרק את התיקייה, וחזר עם תשובה - **בלי שכתבנו לולאת כלים ובלי שמימשנו את הכלים**. הפונקציה `query` היא **אסינכרונית** ומחזירה זרם הודעות (async iterator), ולכן היא רצה בתוך `async def` ועם `asyncio.run`.

אותו סוכן ב-TypeScript:

```typescript
import { query } from "@anthropic-ai/claude-agent-sdk";

for await (const message of query({
  prompt: "אילו קבצים יש בתיקייה הזאת?",
  options: { allowedTools: ["Bash", "Glob"] },
})) {
  if ("result" in message) console.log(message.result);
}
```

שימו לב שב-TypeScript שם השדה הוא `allowedTools` (camelCase), בעוד שבפייתון הוא `allowed_tools` (snake_case) - זה ההבדל המוסכם בין השפות.

## הכלים המובנים - built-in tools

הכוח האמיתי של ה-Agent SDK הוא סט הכלים שמגיע מובנה. אלה בדיוק הכלים שמפעילים את Claude Code:

| כלי - tool | מה הוא עושה |
| --- | --- |
| `Read` | קריאת קובץ מהתיקייה |
| `Write` | יצירת קובץ חדש |
| `Edit` | עריכה מדויקת של קובץ קיים |
| `Bash` | הרצת פקודות מעטפת, סקריפטים, git |
| `Glob` | מציאת קבצים לפי תבנית (`**/*.py`) |
| `Grep` | חיפוש בתוכן קבצים עם regex |
| `WebSearch` | חיפוש באינטרנט למידע עדכני |
| `WebFetch` | הבאת תוכן של דף אינטרנט וניתוחו |

בזכות הכלים האלה, הסוכן יכול לקרוא קוד, לערוך אותו, להריץ בדיקות ולחפש מידע - מיד, בלי שנממש כלום. הנה סוכן שמחפש הערות TODO בקוד ומסכם אותן:

```python
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions


async def main():
    async for message in query(
        prompt="מצא את כל הערות ה-TODO בקוד וצור סיכום",
        options=ClaudeAgentOptions(allowed_tools=["Read", "Glob", "Grep"]),
    ):
        if hasattr(message, "result"):
            print(message.result)


asyncio.run(main())
```

## שליטה בהרשאות - permissions

השדה `allowed_tools` הוא גם מנגנון האבטחה שלנו. הוא קובע אילו כלים מאושרים מראש. אם נבנה סוכן שרק **מנתח** קוד ולא **משנה** אותו, ניתן לו רק כלים לקריאה:

```python
options = ClaudeAgentOptions(
    allowed_tools=["Read", "Glob", "Grep"],   # קריאה בלבד - אין Write/Edit/Bash
)
```

כך אנחנו שולטים בדיוק במה שהסוכן יכול לעשות. יש גם `permission_mode` שקובע כמה הסוכן צריך לבקש אישור - למשל `"acceptEdits"` כדי לאשר אוטומטית עריכות קבצים. עבור פעולות רגישות, ה-SDK יכול לעצור ולבקש אישור אנושי לפני הביצוע.

## יכולות מתקדמות - advanced capabilities

מעבר לכלים המובנים, ה-Agent SDK מביא כמה יכולות חזקות שהיינו צריכים לבנות בעצמנו:

**חיבור לשרתי MCP** - אפשר לחבר את הסוכן למערכות חיצוניות (בסיסי נתונים, דפדפנים, API) דרך פרוטוקול MCP (נלמד עליו בהרצאה הבאה):

```python
options = ClaudeAgentOptions(
    mcp_servers={
        "playwright": {"command": "npx", "args": ["@playwright/mcp@latest"]}
    }
)
```

**תת-סוכנים** - הסוכן הראשי יכול להאציל משימות ממוקדות לתת-סוכנים מיוחדים, שמדווחים בחזרה עם תוצאות. שימושי לפירוק עבודה מקבילית.

**סשנים** - שמירת הקשר בין קריאות. הסוכן זוכר קבצים שקרא וניתוח שביצע. אפשר לתפוס את מזהה הסשן מהקריאה הראשונה ולהמשיך אותו מאוחר יותר:

```python
options = ClaudeAgentOptions(resume=session_id)
```

**hooks** - הרצת קוד מותאם בנקודות מפתח במחזור החיים של הסוכן, למשל רישום כל שינוי בקובץ ל-audit log דרך ה-hook בשם `PostToolUse`.

## מתי להשתמש במה - when to use what

יש כמה דרכים לבנות עם Claude, וכדאי לדעת מתי כל אחת מתאימה:

| אתה רוצה... | השתמש ב... |
| --- | --- |
| שליחת בקשה בודדת, סיווג, סיכום | Client SDK (קריאת API) |
| לולאת כלים עם כלים משלך, בשליטה מלאה | Client SDK + מריץ הכלים |
| סוכן שעובד על קבצים ופקודות בתשתית שלך | **Agent SDK** |
| סוכן מנוהל שאנתרופיק מריצה בתשתית שלה | Managed Agents (שירות מנוהל) |

ה-Agent SDK מבריק כשאתה רוצה סוכן קוד/מערכת-קבצים "עם סוללות כלולות" שרץ **בתהליך ובתשתית שלך**. אם אתה רק צריך קריאת API אחת - ה-Client SDK פשוט יותר. אם אתה לא רוצה לתחזק את התשתית בכלל - Managed Agents. דרך נפוצה היא לאב-טיפוס עם ה-Agent SDK מקומית, ואז לעבור ל-Managed Agents לייצור.

## דוגמה מלאה - a bug-fixing agent

בואו נחבר הכל לסוכן שמתקן באג. ניתן לו כלים לקריאה, עריכה והרצה, והוא יעשה את כל השאר בעצמו - יקרא את הקובץ, ימצא את הבעיה, יערוך, ואפילו יריץ בדיקה:

```python
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions


async def main():
    async for message in query(
        prompt="מצא ותקן את הבאג ב-calculator.py, ואז הרץ את הבדיקות כדי לוודא.",
        options=ClaudeAgentOptions(
            allowed_tools=["Read", "Edit", "Bash", "Glob", "Grep"],
        ),
    ):
        # מדפיסים כל הודעה כדי לראות את הסוכן חושב ופועל
        print(message)


asyncio.run(main())
```

בהרצה, נראה את הסוכן קורא את הקובץ, מזהה את השורה הבעייתית, עורך אותה, מריץ את הבדיקות, ואם משהו נכשל - מנסה שוב. כל הלולאה הזו, שבנינו ידנית בהרצאה הקודמת, מנוהלת עכשיו לגמרי על ידי ה-SDK.

## סוגי ההודעות - message types

הפונקציה `query` מחזירה **זרם הודעות**, לא תשובה אחת. חשוב להבין מה הזרם מכיל, כי כך "רואים" את הסוכן חושב ופועל בזמן אמת. יש כמה סוגי הודעות:

- הודעת אתחול (init) - נשלחת בתחילת הסשן ומכילה מטא-נתונים, כמו `session_id`.
- הודעות ביניים - טקסט שהמודל מייצר, קריאות לכלים ותוצאותיהן.
- הודעת תוצאה (result) - התוצאה הסופית של המשימה.

אפשר לסנן לפי הסוג. למשל, לתפוס את מזהה הסשן מהודעת האתחול, כדי להמשיך אותו אחר כך:

```python
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions, SystemMessage, ResultMessage


async def main():
    session_id = None

    # שאילתה ראשונה - תופסים את מזהה הסשן
    async for message in query(
        prompt="קרא את מודול האימות",
        options=ClaudeAgentOptions(allowed_tools=["Read", "Glob"]),
    ):
        if isinstance(message, SystemMessage) and message.subtype == "init":
            session_id = message.data["session_id"]

    # ממשיכים עם ההקשר המלא של השאילתה הראשונה
    async for message in query(
        prompt="עכשיו מצא את כל המקומות שקוראים לו",
        options=ClaudeAgentOptions(resume=session_id),
    ):
        if isinstance(message, ResultMessage):
            print(message.result)


asyncio.run(main())
```

בשאילתה השנייה, המילה "לו" מתייחסת למודול האימות מהשאילתה הראשונה - כי המשכנו את אותו סשן דרך `resume`. הסוכן זוכר מה קרא ומה ניתח.

## תת-סוכנים - subagents

יכולת חזקה נוספת היא **תת-סוכנים**: הסוכן הראשי מאציל משימות ממוקדות לסוכנים מיוחדים, שמדווחים בחזרה. מגדירים אותם דרך `agents`, ומוסיפים `Agent` ל-`allowed_tools` כדי לאשר את ההפעלה:

```python
import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions, AgentDefinition


async def main():
    async for message in query(
        prompt="השתמש בסוכן code-reviewer כדי לבקר את הקוד",
        options=ClaudeAgentOptions(
            allowed_tools=["Read", "Glob", "Grep", "Agent"],
            agents={
                "code-reviewer": AgentDefinition(
                    description="מבקר קוד מומחה לאיכות ואבטחה.",
                    prompt="נתח איכות קוד והצע שיפורים.",
                    tools=["Read", "Glob", "Grep"],
                )
            },
        ),
    ):
        if hasattr(message, "result"):
            print(message.result)


asyncio.run(main())
```

לכל תת-סוכן יש הקשר משלו, כלים משלו והנחיות משלו. זה שימושי לפירוק עבודה מקבילית - למשל, סוכן אחד סורק קבצים ותת-סוכן אחר מבקר אותם - וגם לחיסכון בהקשר, כי כל תת-סוכן מתמקד במשימה שלו.

## הגדרות מבוססות קבצים - filesystem config

ה-Agent SDK תומך גם בהגדרות של Claude Code שנשמרות בקבצים. עם ברירת המחדל, ה-SDK טוען אותן מתיקיית `.claude/` בפרויקט ומ-`~/.claude/`:

| יכולת | תיאור | מיקום |
| --- | --- | --- |
| מיומנויות - skills | יכולות מיוחדות שהמודל טוען לפי הצורך | `.claude/skills/*/SKILL.md` |
| זיכרון - memory | הקשר והנחיות לפרויקט | `CLAUDE.md` |
| תוספים - plugins | הרחבות עם כלים, סוכנים ו-hooks | דרך אפשרות `plugins` |

זה אומר שאותן מיומנויות והנחיות שמגדירים ל-Claude Code זמינות אוטומטית גם לסוכן שבונים ב-SDK. אם רוצים להגביל אילו מקורות נטענים, מגדירים `setting_sources` באפשרויות.

## סיכום

בהרצאה הזו הכרנו את ה-Agent SDK - הדרך לבנות סוכן שלם בלי לכתוב את הלולאה בעצמנו:

- **ה-Agent SDK** (`claude-agent-sdk` בפייתון, `@anthropic-ai/claude-agent-sdk` ב-npm) הוא Claude Code ארוז כספרייה: כלים מובנים, לולאת סוכן, ניהול הקשר, הרשאות וסשנים.
- ההבדל מ**ה-Client SDK**: שם אנחנו כותבים את לולאת הכלים ומממשים כלים; כאן Claude מנהל את הלולאה עם כלים מוכנים.
- הפונקציה המרכזית היא `query(prompt, options)`, וההגדרות עוברות דרך `ClaudeAgentOptions` (פייתון) או אובייקט `options` (TypeScript). היא אסינכרונית ומחזירה זרם הודעות.
- **הכלים המובנים** - `Read`, `Write`, `Edit`, `Bash`, `Glob`, `Grep`, `WebSearch`, `WebFetch` - נותנים לסוכן יכולת לפעול מיד, בלי מימוש.
- **הרשאות** נשלטות דרך `allowed_tools`: נותנים רק את הכלים הדרושים, ומשתמשים ב-`permission_mode` לאישורים.
- **יכולות מתקדמות**: חיבור לשרתי MCP, תת-סוכנים, סשנים ו-hooks.
- **בחירה נכונה**: קריאת API בודדת - Client SDK; סוכן על תשתית שלך - Agent SDK; סוכן מנוהל - Managed Agents.

בהרצאה הבאה נסגור את המעגל - נלמד לבנות **שרת MCP** משלנו, שחושף כלי מותאם אישית שאפשר לחבר גם ל-Agent SDK, גם ל-Claude Code, וגם לכל מארח MCP אחר.
