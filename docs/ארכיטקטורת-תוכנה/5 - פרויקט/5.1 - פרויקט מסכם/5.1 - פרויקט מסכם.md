## פרויקט מסכם: TaskFlow 2.0

בפרויקט המסכם נחבר את כל מה שלמדנו בקורס ונשדרג את מערכת TaskFlow לארכיטקטורה מקצועית ומלאה.

---

## המשימה

קחו את קוד ה-TaskFlow מהפרויקט ההתחלתי (פרק 0.2) ושדרגו אותו לפי כל העקרונות שלמדנו.

---

## דרישות חובה

### 1. ארכיטקטורה נקייה

מבנה קבצים:
```
taskflow/
    entities/
        task.py       (Task, TaskStatus - פייתון טהור)
        user.py
        project.py
    use_cases/
        task_use_cases.py    (CreateTask, CompleteTask, AssignTask)
        project_use_cases.py
    adapters/
        sqlite_task_repo.py
        email_notifier.py
    main.py
```

### 2. עקרונות SOLID

- כל מחלקה - אחריות אחת
- Dependency Injection - services מקבלים repositories בconstructor
- Open/Closed - הוספת notification channel חדש ללא שינוי קוד קיים

### 3. תבניות עיצוב

חייבים לשלב לפחות שתיים מהתבניות הבאות:
- **Factory** - ליצירת notification channels
- **Observer** - להפצת אירועים כשמשימה משתנה
- **Repository Pattern** - הפרדת DB logic
- **Builder** - ליצירת tasks מורכבות

### 4. ארכיטקטורת שכבות

הפרידו ל:
- `routes/` - HTTP handling בלבד
- `services/` - לוגיקה עסקית
- `repositories/` - DB access

---

## דרישות בחירה (בחרו שתיים)

### אפשרות א: Event-Driven

שלבו EventBus פשוט:
- TaskCreated → שליחת מייל + רישום ב-audit log
- TaskCompleted → עדכון סטטיסטיקות
- TaskAssigned → התראה לנמען

### אפשרות ב: CQRS

הפרידו Read/Write:
- Commands: `CreateTask`, `UpdateTask`, `CompleteTask`
- Queries: `GetTask`, `GetDashboard`, `GetOverdueTasks`

### אפשרות ג: עיצוב לסקייל

כתבו מסמך ארכיטקטורה (ARCHITECTURE.md) שעונה:
1. איך TaskFlow יראה עם 1 מיליון משתמשים?
2. מה יהיו ה-bottlenecks הראשונים?
3. מה תוסיפו ראשון (cache? replicas? sharding?)
4. האם תפצלו למיקרוסרביסים? אילו? מתי?

## טיפים

- **התחילו קטן:** אל תנסו לממש הכל בבת אחת. התחילו עם entities, ואז use cases, ואז adapters.
- **בדקו כל שלב:** כתבו בדיקה לכל use case לפני שממשיכים.
- **לא צריך UI:** ה-API בלבד מספיק. Postman לבדיקות ידניות.
- **Git:** עשו commit בסוף כל שלב עם הסבר ברור.

---

## בונוס - לא חובה

- **API Versioning:** `/api/v1/tasks` כך שניתן לעדכן את ה-API בעתיד
- **Rate Limiting:** מניעת flood של בקשות ממשתמש אחד
- **Pagination:** `GET /tasks?page=2&size=20`
- **Full-Text Search:** חיפוש בכותרות ותיאורי משימות
