### פתרון לתרגיל 1: ניהול processes ושימוש ב-`jobs` ו-`bg`

1. הרצת פקודת `sleep 30`:

```bash
sleep 30
```

1. עצרו את הprocess על ידי לחיצה על `Ctrl + Z`. הprocess יעצור ויושם בפריז.

2. הריצו את הפקודה `jobs`:

```bash
jobs
```

תוצאה צפויה:

```
[1]+ 12345  <sleep 30>
```

כאן 12345 הוא ה-PID של הprocess שלכם, והמספר 1 הוא מספר הג'וב.

3. החזירו את הprocess לפעולה ברקע עם הפקודה:

```bash
bg %1
```

4. הריצו שוב את הפקודה `jobs`:

```bash
jobs
```

תוצאה צפויה:

```
[1]+ 12345  <sleep 30>&
```

הג'וב רץ ברקע.

2. הרצת פקודת `sleep 60`:

```bash
sleep 60
```

1. עצרו את הprocess עם `Ctrl + Z`.
2. השתמשו בפקודה `fg %2` כדי להחזיר את הprocess לפעולה בתור הג'וב הראשי:

```bash
fg %2
```

3. הריצו את הפקודה `ps aux`:

```bash
ps aux
```

הprocess `sleep 60` יופיע ברשימה של הprocesses.

### פתרון לתרגיל 2: שליחת אותות לprocesses

1. הריצו את הפקודה `sleep 60`:

```bash
sleep 60
```

1. הריצו `ps aux` כדי למצוא את ה-PID של הprocess:

```bash
ps aux
```

תוצאה צפויה:

```
user     12345  0.0  0.0   1234  5678 ?        S    10:00   0:00 sleep 60
```

ה-PID של הprocess הוא 12345.

2. שלחו לprocess את אות SIGTERM עם הפקודה:

```bash
kill -15 12345
```

3. הריצו שוב את הפקודה `ps aux` כדי לוודא שהprocess לא קיים יותר:

```bash
ps aux
```

הprocess לא יופיע ברשימה.

2. הריצו שוב את הפקודה `sleep 60`:

```bash
sleep 60
```

1. עצרו את הprocess עם `Ctrl + Z`.
2. שלחו לprocess את אות SIGKILL עם הפקודה:

```bash
kill -9 12345
```

3. הריצו שוב את הפקודה `ps aux`:

```bash
ps aux
```

הprocess לא יופיע ברשימה.
