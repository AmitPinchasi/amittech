בהרצאות הבאות נעבור על כל הדרכים שאתם צריכים להכיר כשאתם נתקלים במחשב ווינדוס שאתם רוצים להסלים איתו לsystem (הroot של ווינדוס)
**אם איכשהו הגעתם לפה מבלי לעשות את קורס הווינדוס החינמי שלנו, עשו אותו לפני!**

לפני שאנחנו מתחילים לגעת בהסלמת הרשאות, קודם נלמד איזה פקודות כדאי לנו להריץ (כמו בהרצאת לינוקס) כדי להביא מידע בסיסי על המחשב!

#### **מבוא להעלאת הרשאות ב-Windows**

ווינדוס- Windows היא מערכת הפעלה עם מודל הרשאות הדוק שמנהל את הגישה של משתמשים ומשאבים שונים. לפני שנוכל לנצל חולשות להעלאת הרשאות, חשוב להבין איך מערכת זו פועלת.

ווינדוס משתמשת בטוקנים (Tokens) כדי לנהל הרשאות, וכל משתמש או תהליך פועל עם טוקן שמכתיב את הגישה שלו למערכת. ישנם מספר סוגים של חשבונות מערכת עם רמות הרשאה שונות:

- **משתמש רגיל (Standard User)** – בעל גישה מוגבלת, ללא אפשרות לבצע שינויים קריטיים במערכת.
- משתמש **Administrator** – מאפשר גישה נרחבת למערכת, כולל התקנת תוכנות ושינויי הגדרות, אך לעיתים מוגבל על ידי בקרת חשבון משתמש (UAC).
- משתמש **SYSTEM** – רמת ההרשאה הגבוהה ביותר, המאפשרת שליטה מוחלטת על המערכת.
- משתמשי **Network Service ו-Local Service** – חשבונות מערכת מוגבלים יותר שמשמשים שירותים של Windows.

#### **איך לבדוק את רמת ההרשאות שלנו?**

הפקודה הבאה מציגה את שם המשתמש הנוכחי והרשאותיו:

```powershell
whoami
```

הפקודה הבאה תציג אילו קבוצות אנחנו שייכים אליהן (לדוגמה, Administrators, Users וכו’):

```powershell
whoami /groups
```

רשימת ההרשאות שיש לנו (Privileges) מוצגת כך:

```powershell
whoami /priv
```

אם אחת ההרשאות הללו, כמו `SeImpersonatePrivilege`, מופיעה עם `Enabled`, יש לנו דרך פוטנציאלית להעלאת הרשאות.

#### **מה זה UAC ואיך הוא משפיע עלינו?**

בקרת חשבון משתמש (User Account Control – UAC) מונעת ממשתמשים להפעיל יישומים ברמת Administrator מבלי לקבל אישור. כאשר נפתח `cmd.exe` עם הרשאות רגילות, הוא לא יוכל לבצע פעולות שדורשות גישה למערכת הקבצים או לרישום (Registry).  
ניתן לבדוק אם אנו פועלים תחת UAC עם:

```powershell
whoami /priv | findstr "Elevated"
```

או לנסות לבדוק הרשאות כתיבה בתיקיות מסויימות עם הכלי icacls

בדקו איזה הרשאות יש לכם לתקייה C:\
```powershell
icacls C:\
```

שיטה להסלמה: לראות אם יש לנו גישה לכתוב בתקייה "C:\Windows\System32"

```powershell
icacls C:\Windows\System32
```
