## מחקר אפליקציות פלאטר - Flutter App Research
פלאטר- **Flutter** היא פריימוורק של Google לפיתוח אפליקציות מובייל בשפת **Dart**.
אפליקציות Flutter הולכות ונהיות יותר ויותר פופולריות, ויש להן מאפיינים ייחודיים שמשפיעים על איך שאנחנו חוקרים אותן.

הדבר החשוב ביותר שצריך לדעת - קוד Dart בFlutter מקומפל ל**קוד native** (ARM assembly).
זה אומר שלא נראה java, לא נראה javascript, ולא נראה bytecode - נראה קוד מכונה, בדומה לספריית so רגילה.

## איך לזהות אפליקציית Flutter - How to Identify
כשפותחים APK של אפליקציית Flutter, נראה:
- בתיקיית lib שני קבצים חשובים:
  - ה `libflutter.so` - מנוע הFlutter עצמו
  - ה`libapp.so` - הקוד של האפליקציה (כל הלוגיקה)
- בjadx כמעט אין קוד java מעניין, רק wrapper מינימלי שמפעיל את מנוע הFlutter
- בmanifest שם הactivity הראשי הוא בדרך כלל `io.flutter.embedding.android.FlutterActivity` או יורש ממנו

## מבנה הAPK
בתוך APK של Flutter נמצא:
```
lib/arm64-v8a/libflutter.so   # מנוע Flutter (לא מעניין אותנו)
lib/arm64-v8a/libapp.so       # הקוד של האפליקציה
```
הקובץ `libapp.so` מכיל **Dart snapshot** - זה לא so רגיל, אלא פורמט ייחודי של Dart שמכיל את כל הקוד המקומפל והנתונים של האפליקציה.

## למה Flutter קשה למחקר
ספרית Flutter היא אחת הפריימוורקים הקשות ביותר למחקר, מכמה סיבות:
1. **הקוד מקומפל לnative** - אין bytecode ביניים, הקוד הוא assembly ישיר
2. **פורמט snapshot ייחודי** - הפורמט משתנה בין גרסאות של Dart, אז כלים צריכים להתעדכן כל הזמן
3. **אין metadata כמו בIL2CPP** - בניגוד לUnity, אין קובץ נפרד עם שמות הפונקציות
4. **לא עובר דרך java** - בניגוד לReact Native, Flutter לא משתמש בbridge לjava, אז לא ניתן לתפוס קריאות ברמת הjava

## כלים למחקר - Research Tools

### הכלי blutter
הכלי העיקרי והעדכני ביותר למחקר Flutter:
https://github.com/worawit/blutter

הוא עובד על ידי קומפילציה מחדש של Dart runtime בגרסה שתואמת לאפליקציה, ואז משתמש במידע הזה כדי לנתח את הsnapshot.
```bash
python3 blutter.py path/to/lib/arm64-v8a out_dir
```
הכלי מייצר:
- קובץ עם כל המחלקות, המתודות והשדות עם שמות מלאים
- סקריפט IDA שמסמן את כל הפונקציות בבינארי
- מידע על אובייקטים ומחרוזות

כרגע blutter תומך רק בarm64.

### הכלי reFlutter
כלי שעובד בגישה אחרת - הוא מחליף את מנוע הFlutter באפליקציה בגרסה מותאמת שמדפיסה מידע בזמן ריצה:
https://github.com/nicolo-ribaudo/hermes-dec

```bash

```
הכלי יוצר APK חדש עם מנוע Flutter מותאם. כשמריצים את האפליקציה, היא מדפיסה לterminal את שמות כל הפונקציות שנקראות בזמן ריצה.
זה שימושי מאוד כדי להבין מה קורה כשמבצעים פעולות מסוימות באפליקציה.

## מה כן אפשר לעשות - What We Can Do
למרות הקושי, יש כמה דברים שאפשר לעשות עם אפליקציות Flutter:

### ניתוח תעבורת רשת - Network Traffic
אפליקציות Flutter לא משתמשות בcertificate store של המערכת כברירת מחדל. הן משתמשות בcertificate store משלהן שמקומפל לתוך `libflutter.so`.
זה אומר שגם אם נתקין את הCA של burp על המכשיר, Flutter לא יסמוך עליו.

הפתרון הוא לפאצ'פצ׳ את `libflutter.so` כדי לבטל את בדיקת הSSL.
הכלי reFlutter עושה את זה אוטומטית - הוא מחליף את libflutter.so בגרסה שמקבלת כל סרטיפיקט.

### חיפוש מחרוזות - String Search
גם בלי דיקומפילציה מלאה, אפשר לחפש מחרוזות בlibapp.so:
```bash
strings libapp.so | grep -i "api\|token\|secret\|password\|http"
```
לפעמים מוצאים כתובות API, מפתחות, או מידע רגיש אחר.

### פרידה עם Flutter - Frida with Flutter
אפשר להשתמש בfrida כדי לעשות hook לפונקציות native בlibapp.so.
אחרי שמשתמשים בblutter כדי לקבל את כתובות הפונקציות:
```javascript
var libapp = Module.findBaseAddress("libapp.so");
var funcOffset = 0xABCDEF; // כתובת מblutter

Interceptor.attach(libapp.add(funcOffset), {
    onEnter: function(args) {
        console.log("Function called");
    },
    onLeave: function(retval) {
        console.log("Returned: " + retval);
    }
});
```

## סיכום התהליך - Workflow Summary
1. זהי שזו אפליקציית Flutter (חפשי libflutter.so ו-libapp.so)
2. הריצי blutter על תיקיית הlib כדי לקבל שמות מחלקות ומתודות
3. לניתוח תעבורת רשת - השתמשי בreFlutter כדי לעקוף SSL
4. חפשי מחרוזות מעניינות בlibapp.so
5. השתמשי בfrida עם Interceptor לhook ברמת הnative
6. לניתוח עמוק - פתחי את libapp.so בGhidra עם הסקריפט מblutter

ספרית Flutter היא הפריימוורק הקשה ביותר למחקר מבין השלוש שלמדנו, אבל עם הכלים הנכונים אפשר עדיין להוציא מידע חשוב.
