## Obfuscation אבפוסקציה

### מהי Obfuscation

הObfuscation היא טכניקה שמטרתה להקשות על קריאה והבנה של קוד תוכנה.  
באנדרואיד משתמשים בה בעיקר כדי להגן על קוד האפליקציה מפני חוקרים כמונו

האופוסקציה לא מונעת לחלוטין מחקר של האפליקציה, אך היא הופכת את התהליך למורכב יותר.
## איך נראה קוד ללא Obfuscation

לדוגמה קוד רגיל:
```java
public class LoginManager {

    public boolean validatePassword(String password){
        if(password.equals("123456")){
            return true;
        }
        return false;
    }

}
```

כאשר נפתח APK כזה ב־jadx נראה קוד ברור עם שמות משתנים ופונקציות מובנים.

## איך נראה קוד לאחר Obfuscation

לאחר אופוסקציה הקוד יכול להיראות כך:
```java
public class a {

    public boolean a(String b){
        if(b.equals("123456")){
            return true;
        }
        return false;
    }

}
```

שמות המחלקות, הפונקציות והמשתנים הוחלפו לשמות קצרים וחסרי משמעות.

## מה משתנה באופוסקציה
 **שינוי שמות (Renaming)**
- מחלקות classes
- מתודות methods
- משתנים variables
- חבילות packages

לדוגמה:
```text
com.company.payment.PaymentManager
```

הופך ל:
```text
a.b.c.a
```

 **שינוי מבנה הקוד**
לפעמים הכלים משנים את מבנה הקוד כדי להקשות על קריאה.
לדוגמה:
```java
if(x == 1){
    doSomething();
}
```

יכול להפוך ל:
```java
switch(x){
    case 1:
        doSomething();
        break;
}
```

**הString Obfuscation**
לעיתים גם מחרוזות מוסתרות.
לדוגמה במקום:
```java
String url = "https://api.server.com/login";
```
נראה משהו כזה:
```java
String url = decrypt("ajskd8123ks");
```

המחרוזת האמיתית מפוענחת בזמן ריצה.
