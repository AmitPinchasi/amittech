
## תמיכה במספר פעילויות
לפעמים, את תרצי לפתח אפליקציות עם כמה סוגי פעילויות. 
דמייני שיש לך אפליקציה עם מסך בית, מסך הגדרות, מסך התחברות וכו.
באנדרואיד נוכל לממש מספר פעילויות שונות, וכדי לעבור בין פעילויות- (למשל בלחיצת כפתור) פעילות צריכה לשלוח ״intent" לפעילות אחרת
ה**Intent (כוונה/רצון)** הוא מנגנון שמאפשר ל־Activity (פעילות) אחת לתקשר עם Activity (פעילות) אחרת.

## Explicit intent
אפשר לחשוב על Intent כעל "בקשה לבצע פעולה".
כאשר פעילות אחת, רוצה לפתוח פעילות אחרת- היא שולחת לה intent (בקשה)
הנה דוגמה ליצירת intent לצורך פיתחת activity אחר באפליקציה
```kotlin
val intent = Intent(this, SecondActivity::class.java)
startActivity(intent)
```
למעשה, אפשר להשתמש בintent-ים כדי לפתוח פעילויות **באפליקציות אחרות**, לא רק מסך באפליקציה שלך.
דמייני ולמשל, את רוצה לפתח פעילות שפותחת את אפליקצית המצלמה בטלפון, (למשל כדי לשים תמונת פרופיל חדשה באפליקציה)- נוכל לבצע intent שפותח את הmain activity של אפליקציית המצלמה.
```kotlin
val intent = Intent().setClassName("com.camera", "com.camera.mainactivity.MainActivity")
startActivity(intent)
```

ואפשר אפילו גם להעביר מידע בintent-ים.
דמייני, ונרצה לפתוח activity של הדפדפן- אבל נרצה להעביר לפעילות גם איזה url נרצה שהוא יפתח בשבילנו.
בשביל זה ישנה אפשרות גם להעביר מידע בintent-ים.
```kotlin
val intent = Intent().setClassName("com.google.chrome", "com.google.chrome.mainactivity.MainActivity")
intent.data = Uri.parse("https://www.google.com")
startActivity(intent)
```
בדוגמה הזו, הפעילות שלנו תפתח את גוגל כרום עם הurl שציינו.
בקוד של הפעילות הזו בכרום, יהיה קטע קוד שידע לפרסר המידע הזה.
```kotlin
fun OnCreate(){
	val intent = getIntent()
	val uri = intent.data
	
	val webview = WebView()	
	webview.loadUrl(uri)
}
```
בנוסף, כאשר אנחנו מעבירים מידע בintent-ים, אנחנו יכולים גם להעביר ״extras"ֿ, אקסטות- הן מילון עם מפתחות וערכים שאפשר להעביר בintent-ים.
לדוגמה- בדוגמת הדפדפן.
```kotlin
val intent = Intent().setClassName("com.google.chrome", "com.google.chrome.mainactivity.MainActivity")
intent.data = Uri.parse("https://mamicademy.com")
intent.putExtra("method", "POST")
intent.putExtra("data", '{"username": "shir", "password": "AMITP123"}')
startActivity(intent)
```
בקוד של chrome תצפי ל-
```kotlin
fun OnCreate(){
	val intent = getIntent()
	val uri = intent.dataֿ
	val method = intent.getExtra("method")
	val _data = intent.getExtra("data")
	
	val webview = WebView()
	webview.setMethod(method)
	webview.setData(_data)
	webview.loadUrl(uri)
}
```

דומאות שבהן נשתמש בintent-ים:
- לפתוח מסך אחר באפליקציה שלנו
- להעביר נתונים בין מסכים באפליקציה
- לפתוח דפדפן (פעילות חיצונית)
- לפתוח מצלמה (פעילות חיצונית)
- לשתף טקסט (פעילות חיצונית)

בגדול יש שני סוגים עיקריים של intent-ים:
1. ה**Explicit Intent** – פונה ישירות ל־Activity מסוימת. (עד כו דיברנו על עליו)
2. ה**Implicit Intent** – מבקש לבצע פעולה כללית (action), ואנדרואיד מחליטה איזו אפליקציה תטפל בזה.

## Implicit Intent
נחזור לדוגמה שלנו של הדפדפן.
נניח שאנחנו רוצים לפתוח אתר, אבל אנחנו לא יודעים מה הדפדפן הדיפולטי של המכשיר, או איזה activity צריך להריץ.
אפשר במקום לעשות specify לactivity, להגיד איזו פעולה ״action" את רוצה שתקרה.
למשל כדי לפתוח דפדפן נוכל לעשות ככה:
```kotlin
val intent = Intent(Intent.ACTION_VIEW)
intent.data = Uri.parse("https://www.google.com")
startActivity(intent)
```

כאן:
- אנחנו לא מציינים Activity מסוימת
- אנחנו אומרים: אני רוצה לבצע פעולה מסוג VIEW
- אנדרואיד תבחר אפליקציה מתאימה על פי הפעולה והמידע שהעברנו (לרוב דפדפן)

 דוגמה- פתיחת חייגן
```kotlin
val intent = Intent(Intent.ACTION_DIAL)
intent.data = Uri.parse("tel:0501234567")
startActivity(intent)
```
זה יפתח את מסך החיוג עם המספר מוכן.

מכאן עולה השאלה, כיצד android יודע לשייך action לactivity?
למעשה בmanifest של אפליקציה את יכולה לציין את כל הactions שactivity מסוים יכול לקבל. (נראה את זה בהרצאה הבאה)
