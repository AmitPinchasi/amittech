## ✅ פתרונות – תרגול כתיבת קוד למעבד הפשוט שלנו

---

### שאלה 1 – חיבור והצגה בפלט

```asm
mov 8 -> register 1  
mov 6 -> register 2  
add                 ; adds register 1 and register 2, stores in register 3  
out register 3         ; displays 14
```

 **תוצאה בפלט:** `14`

---

### שאלה 2 – שמירה בזיכרון

```asm
in -> register 1        ; input from user  
wrt register 1 -> RAM[20]  ; write to memory  
ldr RAM[20] -> register 2  ; read from memory  
out register 2          ; output the value
```

 **תוצאה בפלט:** תלוי בקלט מהמשתמש (למשל, אם המשתמש הקיש 7 – יודפס 7)

---

### שאלה 3 – חיסור ושמירה

```asm
mov 25 -> register 1  
mov 10 -> register 2  
sub                 ; 25 - 10 = 15, stored in register 3  
wrt register 3 -> RAM[5]  
ldr RAM[5] -> register 1  
out register 1
```

**תוצאה בפלט:** `15`

---

### שאלה 4 – העתקה בין אוגרים

```asm
mov 4 -> register 1  
mov register 1 -> register 3  
out register 3
```

**תוצאה בפלט:** `4`

---

### שאלה 5 – תוכנה שמשכפלת קלט

```asm
in -> register 1  
out register 1
```

 **תוצאה בפלט:** הערך שהמשתמש הזין.

---

### שאלה 6 – הבנת מהלך ההרצה

```asm
mov 6 -> register 1  
mov 9 -> register 2  
add  
mov register 3 -> register 1  
out
```

ניתוח של כל שלב:

|שלב|פעולה|תוצאה בכל אוגר|
|---|---|---|
|1|`mov 6 -> אוגר 1`|א1=6|
|2|`mov 9 -> אוגר 2`|א2=9|
|3|`add`|א3=6+9=15|
|4|`mov אוגר 3 -> אוגר 1`|א1=15|
|5|`out`|מוציא את א1: 15|

**תוצאה בפלט:** `15`

---

### שאלה 7 – חיבור, שמירה בזיכרון וקריאה

```asm
in -> register 1  
in -> register 2  
add  
wrt register 3 -> RAM[15]  
ldr RAM[15] -> register 3  
out register 3
```

**תוצאה בפלט:** סכום שני המספרים שהמשתמש הזין.

לדוגמה: אם המשתמש הקיש 3 ואז 7 → תוצאת הפלט תהיה `10`.

---

### שאלה 8 – שאלה פתוחה: מתי המעבד מפסיק לרוץ?

**תשובה:**  
המעבד מריץ הוראות לפי מה שמצביע עליו אוגר ה־IP (Instruction Pointer).  
ברגע שה־IP מגיע לכתובת ש**שווה לאוגר הסגמנט של המשתנים (DATA)** – המעבד מבין שזה סוף סגמנט הקוד ומפסיק להריץ פקודות.

לדוגמה:

- אם **אוגר הקוד (code segment)** = 1
    
- ואם **DATA = 6**  
    → אז המעבד יריץ רק את הכתובות 1–5 ב־RAM.
    
 **למה זה חשוב?** כי ככה המעבד לא מנסה להריץ "קוד" שנמצא בתוך משתנים בטעות.
