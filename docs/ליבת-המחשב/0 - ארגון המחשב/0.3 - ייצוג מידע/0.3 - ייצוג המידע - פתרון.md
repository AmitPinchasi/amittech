## ✅ פתרון – ייצוג והמרה של מספרים בינאריים

---

### חלק 1: זיהוי ייצוג בינארי

1. **כמה מספרים שונים ניתן לייצג באמצעות:**
    

|ביטים|נוסחה|מספר אפשרי|
|---|---|---|
|2|2²|4|
|4|2⁴|16|
|8|2⁸|256|
|16|2¹⁶|65536|

---

2. **לאיזה מספר עשרוני מתאים כל אחד מהמספרים הבינאריים הבאים?**
    

|מספר בינארי|חישוב|מספר עשרוני|
|---|---|---|
|0001|1|1|
|0101|4 + 1|5|
|1000|8|8|
|1101|8 + 4 + 0 + 1|13|
|1111|8 + 4 + 2 + 1|15|

---

3. **השוואת מספרים בינאריים:**
    

- א. `1011` (11) > `0111` (7) 
    
- ב. `1000` (8) > `0110` (6) 
    
- ג. `1101` (13) < `1110` (14)

### חלק 4: ביטים באוגרים

1. **אם לאוגר יש 3 ביטים** → 2³ = **8 מספרים שונים**  
    → המספר הכי גבוה שנוכל לייצג הוא **7** (כי מתחילים מ־0)
    
2. **כדי לשמור את 200**:  
    נבדוק מתי 2ⁿ ≥ 200  
    → 2⁷ = 128 
    → 2⁸ = 256 
    → צריך **לפחות 8 ביטים**
    
3. **המספר הכי גבוה עם 5 ביטים**:  
    → 2⁵ = 32 מספרים  
    → הגבוה ביותר הוא 31
    

---

### חלק 5: חשיבה והסבר

1. **למה מחשבים לא עובדים בבסיס 10?**  
    מחשבים בנויים מרכיבים חשמליים, והם "מבינים" רק שני מצבים: **זרם/אין זרם**, שזה מקביל ל־**1 ו־0**. לכן, הכי פשוט עבורם לעבוד בבסיס בינארי.
    
2. **אם נוסיף ביט אחד לאוגר של 4 ביטים (נעבור ל־5 ביטים):**
    
    - 2⁴ = 16
        
    - 2⁵ = 32  
        → נוספו **16 מספרים אפשריים חדשים**!
