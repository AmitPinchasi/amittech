בתרגול הבא נרים רשת active directory וירטואלית בתוך המחשב שלנו.
נרים מחשבים וירטואלים במחשב שלנו, ונחבר אותם לרשת משל עצמם, ושם נרים רשת דומיינית.

# מדריך להקמת סביבה וירטואלית עם VMware ו-Active Directory

## הורדת VMware Workstation Player (Community Edition)
זוהי פלטפורמת הוירטואלזציה שנשתמש בה כדי להרים את המכונות הוירטואליות.

1. היכנס לאתר הרשמי של **VMware Workstation Player** (גרסה חינמית לשימוש אישי).
2. הורד את התוכנה, שימו לב להוריד את הגרסא החינמית - הcommunity edition.
3. התקן את התוכנה לפי ההוראות שעל המסך.

## יצירת מכונה וירטואלית עם Windows Server

1. פתח את **VMware Workstation Player** ולחץ על **Create a New Virtual Machine**.
2. בחר **Installer disc image file (ISO)** ובחר את קובץ ה-ISO של Windows Server (ניתן להוריד מ-Microsoft Evaluation Center).
3. לחץ **Next** ובחר שם ותיקייה למכונה הווירטואלית.
4. הגדר את המשאבים:
    - **מעבד:** לפחות 2 ליבות.
    - **זיכרון (RAM):** לפחות 4GB (מומלץ 8GB ומעלה).
    - **אחסון:** לפחות 40GB.
5. בחר **Network Adapter** והגדר אותו כ- "**Host-Only**" (להתחברות למכונה השנייה בהמשך).
6. השלם את ההתקנה והפעל את המכונה הווירטואלית.

## התקנת Windows Server והגדרת DC

1. התקן את Windows Server כרגיל.
2. לאחר ההתקנה, פתח **Server Manager** ולחץ על **Add roles and features**.
3. בחר **Active Directory Domain Services (AD DS)** ולחץ **Next**.
4. אשר את ההתקנה ואתחל את השרת.
5. לאחר האתחול, פתח **Server Manager** ולחץ על **Promote this server to a domain controller**.
6. בחר **Add a new forest**, תן שם לדומיין (לדוגמה: **amittech.local**), ולחץ **Next**.
7. בחר **Forest Functional Level** כ- **Windows Server 2016 או 2019** והשאר את ברירות המחדל.
8. אשר את ההגדרות והשלם את התהליך. לאחר מכן, השרת יתאפס מחדש.

## יצירת מכונה וירטואלית נוספת עם Windows 10/11

1. צור מכונה וירטואלית חדשה באותו אופן כמו מקודם, אבל עם **Windows 10/11**.
2. בעת הגדרת ה-Network Adapter, בחר **Host-Only** (כדי שהמכונה תוכל לתקשר עם הדומיין).
3. התקן את מערכת ההפעלה כרגיל.

## חיבור שתי המכונות לאותה רשת ב-VMware

1. פתח את **VMware Workstation Player** ובדוק שהרשת של שתי המכונות מוגדרת כ:
    - הגדרת **Host-Only** (יוודא ששני המכונות יכולות לתקשר אחד עם השניה).
2. בדוק את כתובת ה-IP של השרת עם הפקודה `ipconfig /all` ב-Command Prompt.
3. שים לב ששני המחשבים יכולים לתקשר עם ping בניהם.
4. במחשב הWindows 10/11, פתח **Network Settings**, לחץ על **Change adapter settings**, והגדר **Preferred DNS** לכתובת ה-IP של ה-DC.

## הכנסת מחשב Windows לדומיין

1. במחשב הווירטואלי עם Windows 10/11, פתח **System Properties** (לחיצה ימנית על "This PC" -> Properties -> Advanced system settings).
2. לחץ על **Change Settings** -> **Change...**.
3. בחר **Domain**, הזן את שם הדומיין (לדוגמה: **amittech.local**), ולחץ **OK**.
4. כאשר תתבקש להכניס אישורים, הזן את שם המשתמש והסיסמה של מנהל הדומיין (Administrator).
5. לאחר החיבור בהצלחה, הפעל מחדש את המחשב.

## יצירת משתמש דומייני ב-Active Directory

1. במכונת ה-DC, פתח **Active Directory Users and Computers** (`dsa.msc`).
2. עבור ל- **Users**, לחץ לחיצה ימנית ובחר **New -> User**.
3. הגדר שם משתמש (לדוגמה: **user1**) והגדר סיסמה.
4. ודא שהמשתמש אינו נדרש לשנות סיסמה בכניסה הראשונה (אלא אם רוצים אחרת).

## התחברות למחשב עם משתמש הדומיין

1. ב-Windows 10/11, התנתק מהמשתמש הנוכחי ולחץ **Switch User**.
2. התחבר עם המשתמש החדש שנוצר (`amittech\user1`).
3. אם הכל מוגדר נכון, המחשב יאמת את המשתמש מול ה-DC ותוכל להתחבר בהצלחה!

### **סיכום**

כעת יש לך סביבה וירטואלית עם שרת Windows שמנהל דומיין, ומחשב לקוח שמחובר אליו. תוכל להוסיף משתמשים נוספים, להגדיר Group Policies ולבצע ניהול רשת מתקדם.
