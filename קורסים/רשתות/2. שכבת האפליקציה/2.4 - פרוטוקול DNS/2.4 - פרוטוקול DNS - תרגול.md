### תרגול: פרוטוקול DNS - מערכת ניהול שמות דומיינים

#### חלק 1: הבנת מבנה ה-DNS

1. **המרת שם דומיין לכתובת IP:**

- נסה להמיר את שם הדומיין `www.example.com` לכתובת IP באמצעות כלי `nslookup`. השתמש בפקודה:

```bash
nslookup www.example.com
```

2. **בקשות DNS:**

- שלח בקשה מסוג **A** עבור הדומיין `www.google.com` וכתוב את כתובת ה-IP שהשרת מחזיר.
- שלח בקשה מסוג **PTR** עם כתובת ה-IP `8.8.8.8` וכתוב את הדומיין המתקבל.
- שלח בקשה מסוג **MX** עבור הדומיין `google.com` וכתוב את ה-IP של שרת המייל המתקבל.
3. **הבנת החלוקה לאזורים (Zones):**

- הסבר איך מבוצעת חלוקה של שמות הדומיינים לאזורים ותחומים, ומהי המטרה של החלוקה הזו.
- כיצד כל שרת DNS אחראי על תחום מסוים?

#### חלק 2: עבודה עם קובץ Hosts

1. **הגדרת רשומת DNS אישית:**

- פתח את קובץ ה-hosts במחשב שלך והוסף את השורה הבאה:

```plaintext
127.0.0.1 mytest.local
```

- לאחר מכן, פתח דפדפן והזן את כתובת הדומיין `mytest.local` והאם אתה מגיע לכתובת ה-IP המקומית שלך.
2. **ניקוי מטמון ה-DNS:**

- השתמש בפקודה כדי לנקות את מטמון ה-DNS במחשב שלך:

```bash
ipconfig /flushdns
```

- בדוק את מטמון ה-DNS לאחר מכן בעזרת:

```bash
ipconfig /displaydns
```
