## מחקר אפליקציות הרמס - Hermes App Research
ספריית **React Native** היא פריימוורק לפיתוח אפליקציות מובייל בjavascript.
אפליקציות React Native פופולריות מאוד - אפליקציות כמו Facebook, Instagram, Discord ועוד רבות בנויות עליה.

מנוע **Hermes** הוא מנוע javascript שMeta (פייסבוק) פיתחו במיוחד לReact Native.
במקום להריץ javascript רגיל, Hermes מקמפל את הjavascript ל**bytecode** - מה שמשפר ביצועים אבל גם מקשה על מחקר, כי אנחנו לא יכולים פשוט לקרוא את הקוד כjavascript.

## איך לזהות אפליקציית React Native - How to Identify
כשפותחים APK של אפליקציית React Native, נראה כמה סימנים מזהים:
- בjadx נראה references לcom.facebook.react
- בmanifest שם הactivity הראשי הוא בדרך כלל משהו כמו `com.example.app.MainActivity` שיורש מ-`ReactActivity`
- בתיקיית assets יש קובץ שנקרא `index.android.bundle`
- בתיקיית lib יש `libjsc.so` (מנוע javascript רגיל) או `libhermes.so` (מנוע Hermes)

## שני מצבים - Two Modes

### ג'אווהסקריפט רגיל - Plain JavaScript
באפליקציות ישנות יותר, או כאלו שלא משתמשות בHermes, הקובץ `index.android.bundle` הוא פשוט קובץ javascript minified (מוקטן).
במקרה כזה, כל הקוד של האפליקציה בjavascript נמצא שם ואפשר לקרוא אותו.

כדי לבדוק, פרקי את הAPK ותסתכלי על הקובץ:
```bash
apktool d app.apk -o app_extracted
file app_extracted/assets/index.android.bundle
```
אם זה javascript רגיל, הפקודה `file` תראה שזה קובץ טקסט.
אפשר להשתמש בכלי beautifier כדי לקרוא את הקוד בצורה יותר נוחה:
```bash
npx js-beautify app_extracted/assets/index.android.bundle > readable.js
```

### בייטקוד הרמס - Hermes Bytecode
אם האפליקציה משתמשת בHermes, הקובץ `index.android.bundle` הוא bytecode בינארי ולא javascript רגיל.
הפקודה `file` תראה משהו כמו:
```
index.android.bundle: Hermes JavaScript bytecode, version 96
```

אפשר גם לזהות bytecode של Hermes לפי ה-magic bytes בתחילת הקובץ: כל קובץ Hermes מתחיל במחרוזת `c61fbc03` או `1F1903C6`.

## כלים למחקר הרמס - Hermes Research Tools

### הכלי hermes-dec
הכלי העיקרי לדיקומפילציית Hermes bytecode:
https://github.com/nicolo-ribaudo/hermes-dec

הוא מפענח את הbytecode ומחזיר קוד javascript קריא.
```bash
hermes-dec index.android.bundle -o decompiled.js
```
הקוד שמתקבל לא יהיה זהה למקור, אבל יהיה מספיק קריא כדי להבין את הלוגיקה.

שימו לב שhermes-dec צריך לתמוך בגרסת הbytecode הספציפית של הHermes שהאפליקציה משתמשת בו.

### הכלי hbctool
כלי נוסף שמאפשר disassembly ו-assembly של Hermes bytecode:
https://github.com/nicolo-ribaudo/hermes-dec

```bash
hbctool disasm index.android.bundle output_dir
```
הכלי יוצר תיקייה עם קבצי assembly של כל הפונקציות.
היתרון הגדול של hbctool הוא שאפשר גם לערוך את הקוד ולבנות מחדש:
```bash
hbctool asm output_dir patched.bundle
```
כלומר, אפשר לבצע פאצ'ינג על הbytecode.

## תהליך מחקר 
1. פרקי את הAPK ומצאי את `index.android.bundle`
2. בדקי אם זה javascript רגיל או Hermes bytecode
3. **אם javascript רגיל** - הריצי beautifier וקראי את הקוד
4. **אם Hermes** - השתמשי בhermes-dec לדיקומפילציה
5. חפשי בקוד מחרוזות מעניינות - כתובות API, מפתחות, לוגיקה עסקית
6. אם צריך לשנות התנהגות - השתמשי בhbctool לפאצ'ינג, או בfrida לhook

## פרידה עם React Native - Frida with React Native
אפשר להשתמש בfrida כדי לעשות hook לפונקציות javascript בזמן ריצה.
לReact Native יש bridge בין הצד הnative לjavascript, ואפשר לתפוס קריאות שעוברות בbridge הזה.

דוגמה לhook שתופס קריאות API:
```javascript
Java.perform(function() {
    var OkHttpClient = Java.use("okhttp3.OkHttpClient");
    var Builder = Java.use("okhttp3.Request$Builder");

    Builder.url.overload("java.lang.String").implementation = function(url) {
        console.log("Request URL: " + url);
        return this.url(url);
    };
});
```
הרבה אפליקציות React Native משתמשות בספריית OkHttp מאחורי הקלעים לתקשורת רשת, אז אפשר לעשות hook ברמת הjava גם אם הקוד נכתב בjavascript.

## סיכום
- אפליקציות React Native שומרות את הקוד בקובץ `index.android.bundle`
- הקובץ יכול להיות javascript רגיל (קל לקרוא) או Hermes bytecode (צריך כלי דיקומפילציה)
- הכלים hermes-dec ו-hbctool מאפשרים לקרוא ולערוך Hermes bytecode
- אפשר גם לתפוס קריאות ברמת הjava עם frida, כי React Native משתמש בספריות java מאחורי הקלעים
