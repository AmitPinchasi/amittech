## מחקר אפליקציות יוניטי - Unity App Research
חלק מהאפליקציות שנחקור לא נכתבו בjava/kotlin רגיל.
מנוע **Unity** היא מנוע פיתוח משחקים שמאפשר לפתח אפליקציות ומשחקים בשפת C# ולהריץ אותם על מגוון פלטפורמות, כולל אנדרואיד.
כשנפתח APK של אפליקציית Unity בjadx, נראה שכמעט אין קוד java מעניין - כי כל הלוגיקה כתובה בC# ומקומפלת בצורה אחרת.

## איך Unity עובד באנדרואיד - How Unity Works on Android
לUnity יש שני מצבי קומפילציה:

### מונו - Mono
במצב Mono, הקוד של C# מקומפל ל-CIL (Common Intermediate Language) (הbytecode של מיקרוסופט) ונשמר בקבצי DLL. (קבצי ספריות בווינדוס)
הקבצים נמצאים בתוך הAPK בנתיב:
```
assets/bin/Data/Managed/
```
שם נמצא הקובץ החשוב ביותר:
```
Assembly-CSharp.dll
```
הקובץ הזה מכיל את כל הקוד של האפליקציה.

מצב Mono הוא פשוט יחסית לרברסינג כי אפשר להשתמש בכלים רגילים של .NET כדי לפענח את הDLL.

### IL2CPP
מצב IL2CPP הוא המצב הנפוץ יותר כיום.
במצב הזה, Unity לוקח את הקוד CIL ומקמפל אותו הלאה לקוד native (C++), מה שמשפר ביצועים אבל גם מקשה על רברסינג.
ואז מקמפל את הc++ ישירות לso.

הקבצים החשובים במצב IL2CPP נמצאים בAPK:
```
lib/<arch>/libil2cpp.so        # הקוד נייטיב המקומפל
assets/bin/Data/Managed/Metadata/global-metadata.dat  # המטאדאטה
```
הקובץ `libil2cpp.so` מכיל את כל הקוד המקומפל, והקובץ `global-metadata.dat` מכיל מידע על שמות מחלקות, מתודות, שדות ומחרוזות.

## איך לזהות אפליקציית Unity
כשפותחים APK בjadx, יש כמה סימנים שמצביעים על אפליקציית Unity:
- המחלקה הראשית היא `com.unity3d.player.UnityPlayerActivity`
- בmanifest יש references לunity
- בתוך הAPK יש תיקייה `assets/bin/Data/`
- יש קובץ `libunity.so` בתיקיית lib
- יש קובץ `libil2cpp.so` (במצב IL2CPP) או תיקיית `Managed` עם קבצי DLL (במצב Mono)

## כלים למחקר

### מצב Mono - dnSpy / ILSpy
עבור אפליקציות במצב Mono, אפשר להשתמש בכלי דיקומפילציה של .NET:
- הכלי **dnSpy** - כלי מעולה שמראה את הקוד המקורי בC# ואפילו מאפשר עריכה
- הכלי **ILSpy** - כלי דומה, פתוח וזמין גם לmac/linux

התהליך:
1. פרקי את הAPK (עם apktool או שני סיומת לzip)
2. מצאי את `Assembly-CSharp.dll` בתיקיית `assets/bin/Data/Managed/`
3. פתחי את הDLL בdnSpy או ILSpy

תקבלי קוד C# קריא כמעט כמו המקור.

### מצב IL2CPP - Il2CppDumper
עבור אפליקציות במצב IL2CPP, הכלי העיקרי הוא **Il2CppDumper**:
https://github.com/Perfare/Il2CppDumper

התהליך:
1. פרקי את הAPK
2. הריצי את Il2CppDumper עם שני קבצים:
```bash
Il2CppDumper libil2cpp.so global-metadata.dat output_dir
```
3. הכלי ייצר כמה קבצים חשובים:
   - `dump.cs` - קובץ עם כל ההגדרות של המחלקות, המתודות והשדות (בלי המימוש)
   - `script.json` - מיפוי של כתובות הפונקציות בבינארי
   - סקריפט לIDA/Ghidra שמוסיף שמות לפונקציות

הקובץ `dump.cs` הוא המקום הראשון שנסתכל בו. הוא מראה את כל המחלקות והמתודות עם שמות מלאים, סוגי פרמטרים וערכי החזרה - אבל בלי את הקוד עצמו של הפונקציות.

כדי לראות את המימוש של הפונקציות, צריך לפתוח את `libil2cpp.so` בGhidra ולהשתמש בסקריפט שIl2CppDumper ייצר כדי לסמן את כל הפונקציות עם השמות שלהן.

### הכלי Il2CppInspector
כלי חזק יותר שעושה את הכל ביחד:
https://github.com/djkaty/Il2CppInspector

הוא מייצר פלט מפורט יותר וכולל גם תמיכה בדי-אובפוסקציה של metadata מוצפן.

## פרידה עם Unity - Frida with Unity
אפשר להשתמש בfrida גם עם אפליקציות Unity.
במצב IL2CPP, הקוד רץ כnative, אז צריך להשתמש ב-Interceptor של frida (ולא בJava.use):
```javascript
var libil2cpp = Module.findBaseAddress("libil2cpp.so");
var funcOffset = 0x123456; // הכתובת מתוך Il2CppDumper

Interceptor.attach(libil2cpp.add(funcOffset), {
    onEnter: function(args) {
        console.log("called!");
    },
    onLeave: function(retval) {
        console.log("returned: " + retval);
    }
});
```
בinspector כדי לעשות hook, אנחנו מציינים את הכתובת offset של הפונקציה שנרצה לעשות לה hook.
הil2cppdumper יוצר קובץ בשם  `script.json` עם מיפוי של כתובות offset ומה השם של הפונקציה, שם תוכלי לקשר בין הפונקציה שתרצי לעשות לה hook לoffset שלה.

## סיכום התהליך
1. פתחי את הAPK וזהי אם זה Mono או IL2CPP
2. **אם Mono** - שלפי את הDLL ופתחי בdnSpy/ILSpy
3. **אם IL2CPP** - הריצי Il2CppDumper, קראי את dump.cs, ונתחי את libil2cpp.so בGhidra עם הסקריפט שנוצר
4. השתמשי בfrida עם Interceptor לניתוח דינמי
