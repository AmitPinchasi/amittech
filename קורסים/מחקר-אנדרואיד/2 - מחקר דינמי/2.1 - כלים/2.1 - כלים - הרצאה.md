## הכלי adb
הadb או android device bridge מאפשר לנו לבצע המון פעולות מול המכשיר דרך המחשב שלנו, בין אם זה הרצת פקודות (shell), העברת קבצים, ועוד.
כל זה- מעל חיבור usb, או רשתי.

כדי להתחבר למכשיר בadb, הוא צריך להריץ adb server.
למזלנו, האימולטורים של אנדוראיד סטודיו מגיעים עם adb, וגם רצים עם הרשאות root, כדי שנוכל לקבל שליטה מלאה על המכשיר.

פתחי את הטרמינל בandroid studio (הכפתור לזה נמצא בסרגל השמאלי למטה)
בגדול adb כבר צריך להיות מותקן, אבל לפעמים זה לא מוסיף את זה לpath- (במקרה שלי בmac לפחות), בmac זה מותקן ב `Library/Android/sdk/platform-tools/adb/~`
כדי לפתור את זה, אני הוזזתי את adb ל/usr/local/bin (שבmac זה נמצא בpath)
```bash
mv ~/Library/Android/sdk/platform-tools/adb /usr/local/bin
```

## לרוץ כroot
כדי לרוץ כroot, אנחנו צריכים להריץ את הadb server עם הרשאות root.
למזלנו יש פקודה שעושה את זה, "adb root"
כנראה שאם תנסי להריץ את הפקודה, תקבלי: 
״adbd cannot run as root in production builds״
שזה קורה, כאשר adb מנסה לרוץ כroot- אבל בגלל שהאימולטור מריץ image של אנדוראיד שהוא ״מאובטח״, כלומר- בלי root, הוא לא יכול להריץ את adb כroot.

מעבר להרצת פקודות על המכשיר, ל-adb יש עוד כמה יכולות שימושיות.

### העברת קבצים

אפשר להעתיק קבצים מהמחשב למכשיר ולהפך.

העתקת קובץ מהמחשב למכשיר:

```bash
adb push file.txt /sdcard/
```

העתקת קובץ מהמכשיר למחשב:

```bash
adb pull /sdcard/file.txt
```

זה שימושי מאוד כשאנחנו רוצים להביא קבצים מהאפליקציה, לוגים, מסדי נתונים, או קבצי קונפיגורציה.

### התקנת אפליקציות

ניתן להתקין קובץ APK ישירות מהמחשב למכשיר:

```bash
adb install app.apk
```

ואם האפליקציה כבר מותקנת ורוצים לעדכן אותה:

```bash
adb install -r app.apk
```

### צפייה בלוגים של המערכת logcat

אנדרואיד שומר לוגים של המערכת ושל האפליקציות, וניתן לצפות בהם באמצעות:

```bash
adb logcat
```

הלוגים האלו מאוד שימושיים בזמן פיתוח, דיבוג, או מחקר של אפליקציות.
אפשר גם לפלטר לפי סוג הלוג, או האפליקציה שמוציאה אותם.
בנוסף, יש כלי בandroid studio של logcat שמאפשר לנו בצורה הרבה יותר נוחה להסתכל על הלוגים.
באנדרואיד סטודיו, בזמן שהמכשיר רץ- לחצי על הכפתור שנראה כמו חתול בצד שמאל למטה.
![[Pasted image 20260305213302.png]]
ואז יפתח לך החלון עם כל הלוגים של המכשיר
![[Pasted image 20260305222512.png]]
יש המון, המון המון לוגים.
בין היתר broadcast-ים, intent-ים, וסתם לוגים של שירותים שונים ופעילויות שונות.
כמובן שגם פה יש לך אפשרות לפלטר, הפעם בui נוח :)

### טינול (port forwarding)

אפשר לטנל פורט במחשב להיות מחובר לפורט במכשיר:

```bash
adb forward tcp:8080 tcp:8080
```
כך ניתן לגשת לשירות שרץ על המכשיר דרך המחשב.
כלומר, עכשיו כאשר ניגש במחשב לlocalhost:8080 נטונל לפורט 8080 על המכשיר.

### צילום מסך

אפשר לצלם screenshot מהמכשיר:

```bash
adb shell screencap /sdcard/screen.png
```

ואז להוריד את הקובץ למחשב באמצעות `adb pull`.

הפקודות האלו הן רק חלק קטן מהיכולות של adb :)
## הכלי activity manager
כמו שלמדנו, הactivity manager service הוא שירות באנדרואיד שאחראי לנהל את כל הפעילויות באנדוראיד, ובעיקר להעביר intent-ים בין פעילויות.
השירות הזה עובד מעל binder מאחורי הקלעים, ובדיוק כמו שאפליקציות מאחורי הקלעים משתמשות בbinder כדי לתקשר עם השירות הזה (ולהעביר intent-ים בין פעילויות).
אפשר לתקשר עם השירות באמצעות הפקודה `am` (activity manager) בצורה נוחה שמאפשרת לנו באופן יזום לשלוח intent-ים, ולתקשר עם פעילויות ושירותים של אפליקציות.

ניתן להשתמש ב־activity manager באמצעות הפקודה `am`.  
הפקודה הזו שולחת בקשות ל־Activity Manager Service דרך binder.
### הרצת Activity
ניתן להפעיל פעילות של אפליקציה מסוימת באמצעות:
```bash
am start -n com.example.app/.MainActivity
```
הפרמטר `-n` מציין את הפעילות:  
`package/activity`.
### שליחת intent עם נתונים
ניתן לשלוח intent עם מידע נוסף (extras):
```bash
am start -n com.example.app/.MainActivity --es username amit --ei level 5
```
הפרמטרים הנפוצים:
- `--es` מחרוזת (string)
- `--ei` מספר שלם (int)
- `--ez` ערך בוליאני (boolean)

כך אפשר לדמות תקשורת בין אפליקציות.

### שליחת Broadcast
ניתן לשלוח broadcast intent לכל המערכת:
```bash
am broadcast -a com.example.TEST_ACTION
```

ואפשר גם להעביר נתונים:
```bash
am broadcast -a com.example.TEST_ACTION --es msg hello
```
כל BroadcastReceiver שמאזין לaction הזו יקבל את ההודעה.

### הרצת Service
אפשר להפעיל service של אפליקציה:
```bash
am startservice -n com.example.app/.MyService
```

או לעצור אותו:
```bash
am stopservice -n com.example.app/.MyService
```

### סגירת אפליקציה

ניתן להרוג תהליך של אפליקציה:
```bash
adb shell am force-stop com.example.app
```
הפקודה הזו סוגרת את האפליקציה לחלוטין, כולל כל השירותים שלה.

כל הפקודות האלו מאפשרות לנו לדמות אינטראקציות של המערכת עם אפליקציות, לבדוק התנהגות של intent-ים, ולחקור את המנגנון של תקשורת בין רכיבים באנדרואיד.



## שינוי הגדרות
כדי לשנות הגדרות במכשיר אנדרואיד בטרמינל, יש לנו את הפקודות "getprop" ו"setprop" שמאפשרות לנו לגשת להגדרות המערכת של מכשירי אנדרואיד.
הריצי את הפקודה `getprop` בלי פרמטרים כדי לראות את כל ההגדרות של המכשיר.
![[Pasted image 20260305222839.png]]
יש מלא.
```bash
getprop ro.build.version.release
```
מציג את גרסת אנדרואיד של המכשיר.
```bash
getprop ro.build.version.sdk
```
מציג את גרסת ה־SDK של המערכת.
```bash
getprop ro.product.model
```
שם הדגם של המכשיר.
```bash
getprop ro.product.manufacturer
```
היצרן של המכשיר.
```bash
getprop ro.product.brand
```
המותג של המכשיר.
```bash
getprop ro.build.type
```
מציג את סוג ה־build, לדוגמה:
- מצב`user` ( פרודקשן- בלי root)
- מצב`userdebug` (עם root)
- מצב `eng` (האמת שאני לא בטוח)

למעשה getprop/setprop הן הגדרות יותר כלליות של המערכת, ניתן לשנות הגדרות יותר קונקרטיות לאנדרואיד ואפליקציות באמצעות הפקודה settings
בדומה ל-`getprop`, גם הפקודה `settings` מאפשרת לנו לקרוא ולשנות הגדרות של המערכת.  
אבל בניגוד ל-system properties, ההגדרות האלו נשמרות במסד נתונים של Android שנקרא **Settings Provider**.

יש שלושה מרחבים עיקריים של הגדרות:
- `system` הגדרות כלליות של המשתמש
- `secure` הגדרות רגישות יותר של המערכת
- `global` הגדרות מערכת שמשפיעות על כל המשתמשים

### צפייה בהגדרות
כדי לראות את כל ההגדרות בקטגוריה מסוימת:
```bash
adb shell settings list global
```
לדוגמה:
```bash
adb shell settings list system
adb shell settings list secure
```
### קריאת הגדרה ספציפית
אפשר לקרוא ערך מסוים:
```bash
adb shell settings get global http_proxy
```
דוגמה נוספת:
```bash
adb shell settings get global adb_enabled
```

### שינוי הגדרה
ניתן לשנות ערכים במערכת.
לדוגמה, הפעלת ADB debugging:
```bash
adb shell settings put global adb_enabled 1
```
או הגדרת proxy למערכת:
```bash
adb shell settings put global http_proxy 192.168.1.10:8080
```

### דוגמאות להגדרות מעניינות
בדיקת בהירות המסך:
```bash
adb shell settings get system screen_brightness
```
שינוי בהירות המסך:
```bash
adb shell settings put system screen_brightness 200
```
בדיקת זמן כיבוי מסך:
```bash
adb shell settings get system screen_off_timeout
```
בדיקה האם ADB debugging מופעל:
```bash
adb shell settings get global adb_enabled
```

### מחיקת הגדרה
אפשר גם למחוק ערך:
```bash
adb shell settings delete global http_proxy
```

## הכלי burp suit
הכלי burp suit מאפשר לנו לחקור תעבורת http/https.
הכלי מרים שרת http proxy, וכל תעבורה שעוברת דרכו, הוא פותח אותה ומאפשר לנו לראות אותה או אפילו להתערב בה.
הורידי והתקיני burp suit באמצעות:
https://portswigger.net/burp/communitydownload
אחרי שתרימי את burp, תפתחי אותו ותצרי פרויקט חדש.
דיפולטית, הוא ירים שרת proxy על פורט 8080 ב127.0.0.1
אם תלכי לחלון הproxy, ואז settings- תוכלי לראות/לשנות את ההגדרה הזו.

כדי שמכשיר האנדרואיד יעביר את כל הפקטות http שלנו דרך שרת הproxy של burp, נצטרך להגדיר לו קלינט proxy.
בהגדרות הwifi של המכשיר נוכל לעשות את זה בui, ואם אנחנו עצלנים- נוכל גם דרך הטרמינל.
```bash
settings put global http_proxy 192.168.1.10:8080
```
כמובן, שימי לב לשים את הip שלך, (לוודא עם ifconfig) תוודאי שהמכשיר יכול לעשות ping לip שלך.
מעכשיו, כל בקשת http שהמכשיר שלך יעשה, יעבור דרך הproxy של burp ותוכלי לראות אותה בhttp history.
ישנה רק בעיה אחת, כאשר הburp פותח את תעבורת הhttps, הוא בעצם מחליף בתעבורה את הסרטיפיקט שהמכשיר שלנו מנסה להתחבר אילו בסרטיפיקט שלו- אז אנחנו צריכים להתקין ca של burp על המכשיר כדי שהוא יאמת סרטיפיקטים של burp.
כדי לעשות זאת, נפתח את הדפדפן של המכשיר, וניגש לip של המחשב שלנו בפורט 8080, הפרוקסי של burp יודע לקחת אותנו לאתר של burp שדרכו נוכל להוריד את הca.

## הכלי scrcpy
לפעמים, כשאין לנו ui של המכשיר (למשל כשאנחנו לא משתמשים באנדרואיד סטודיו), אפשר לקבל חלון של הui שלו באמצעות הפקודה scrcpy.
הפקודה בודקת איזה מכשירים אנחנו מחוברים אליהם בadb, ומציגה לנו את הui דרך adb מאחורי הקלעים.
נסי להריץ
```bash
scrcpy
```
