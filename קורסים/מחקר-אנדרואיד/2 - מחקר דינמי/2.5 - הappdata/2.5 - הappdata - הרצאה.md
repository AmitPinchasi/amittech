## הנתונים של האפליקציות - App Data
כשאנחנו חוקרים אפליקציה, לא תמיד מספיק לקרוא את הקוד שלה או לעשות hook לפונקציות.
הרבה פעמים מידע חשוב נמצא בקבצים שהאפליקציה שומרת על המכשיר - סיסמאות, טוקנים, הגדרות, מסדי נתונים, ועוד.
בשיעור הזה נלמד איפה אפליקציות אנדרואיד שומרות מידע, ואיך אנחנו יכולים לגשת אליו ולחקור אותו.

## ארגז חול - Sandbox
באנדרואיד, לכל אפליקציה יש תיקייה פרטית שרק היא יכולה לגשת אליה.
התיקייה נמצאת בנתיב:
```
/data/data/<package_name>/
```
למשל, עבור אפליקציה עם שם החבילה `com.example.app`:
```
/data/data/com.example.app/
```
אפליקציות אחרות לא יכולות לגשת לתיקייה הזו, וגם למשתמש רגיל אין גישה אליה.
אבל עם root - אנחנו יכולים לגשת לכל תיקייה שנרצה.

בואו נסתכל על המבנה הטיפוסי של התיקייה:
```bash
adb shell ls /data/data/com.example.app/
```
בדרך כלל נראה תיקיות כמו:
```
shared_prefs/
databases/
cache/
files/
lib/
```
כל תיקייה משמשת לסוג אחר של מידע. נעבור על כל אחת.

## העדפות משותפות - Shared Preferences
הדרך הפשוטה ביותר שאפליקציה שומרת מידע היא באמצעות **SharedPreferences**.
זהו מנגנון של אנדרואיד ששומר זוגות של מפתח-ערך (key-value) בקבצי XML.

הקבצים נמצאים בתיקייה:
```
/data/data/<package_name>/shared_prefs/
```

בואו נסתכל על דוגמה:
```bash
adb shell cat /data/data/com.example.app/shared_prefs/user_prefs.xml
```
נוכל לראות משהו כזה:
```xml
<?xml version='1.0' encoding='utf-8' standalone='yes' ?>
<map>
    <string name="username">amit</string>
    <string name="auth_token">eyJhbGciOiJIUzI1NiJ9...</string>
    <boolean name="is_premium" value="false" />
    <int name="login_count" value="42" />
</map>
```

כאן אפשר לראות:
- שם המשתמש
- טוקן אימות (שאפשר אולי להשתמש בו)
- האם המשתמש פרימיום
- כמה פעמים התחבר

מפתחים עצלנים לפעמים שומרים ב-SharedPreferences מידע רגיש כמו סיסמאות, מפתחות API, טוקנים, ופרטים אישיים - בלי הצפנה.

### עריכת SharedPreferences
אנחנו יכולים גם לערוך את הקבצים האלו.
למשל, אם נרצה להפוך את המשתמש לפרימיום:
```bash
adb shell su -c "sed -i 's/is_premium\" value=\"false/is_premium\" value=\"true/' /data/data/com.example.app/shared_prefs/user_prefs.xml"
```
אחרי העריכה, צריך לסגור את האפליקציה ולפתוח אותה מחדש כדי שהיא תקרא את הערכים החדשים.
```bash
adb shell am force-stop com.example.app
```

## מסדי נתונים - Databases
הרבה אפליקציות משתמשות במסד נתונים מסוג **SQLite** כדי לשמור מידע מורכב יותר.
קבצי מסד הנתונים נמצאים בתיקייה:
```
/data/data/<package_name>/databases/
```

בואו נראה אילו מסדי נתונים יש לאפליקציה:
```bash
adb shell ls /data/data/com.example.app/databases/
```
נוכל לראות קבצים כמו:
```
app.db
app.db-journal
app.db-wal
```
הקבצים עם הסיומות `journal` ו-`wal` הם קבצי עזר של SQLite ולא מעניינים אותנו, הקובץ העיקרי הוא `app.db`.

### חקירה עם sqlite3
אנדרואיד מגיע עם הכלי `sqlite3` שמאפשר לנו לחקור מסדי נתונים ישירות מהטרמינל:
```bash
adb shell sqlite3 /data/data/com.example.app/databases/app.db
```
עכשיו אנחנו בתוך ממשק של SQLite, ואפשר להריץ שאילתות.

להציג את כל הטבלאות:
```sql
.tables
```

להציג את המבנה של טבלה:
```sql
.schema users
```

לראות את כל המידע בטבלה:
```sql
SELECT * FROM users;
```

לחפש מידע ספציפי:
```sql
SELECT * FROM users WHERE username='admin';
```

כדי לצאת מsqlite3:
```sql
.quit
```

### הורדת מסד הנתונים למחשב
לפעמים יותר נוח לעבוד עם מסד הנתונים במחשב.
אפשר להוריד אותו עם adb:
```bash
adb pull /data/data/com.example.app/databases/app.db ./app.db
```
ואז לפתוח אותו עם כלי גרפי כמו **DB Browser for SQLite** שמאפשר לראות ולערוך את המידע בצורה נוחה.

### עריכת מסד הנתונים
אפשר גם לערוך ערכים ישירות:
```bash
adb shell sqlite3 /data/data/com.example.app/databases/app.db "UPDATE users SET role='admin' WHERE username='amit';"
```

## קבצים - Files
תיקיית `files/` משמשת לאחסון קבצים כלליים שהאפליקציה יוצרת.
```bash
adb shell ls /data/data/com.example.app/files/
```
כאן אפשר למצוא כל מיני דברים - קבצי קונפיגורציה, לוגים, קבצים שהורדו, ועוד.
לפעמים אפליקציות שומרות פה קבצי JSON עם מידע רגיש, או קבצים מוצפנים שאפשר לנסות לפענח.

## מטמון - Cache
תיקיית `cache/` מכילה קבצים זמניים שהאפליקציה שומרת.
```bash
adb shell ls /data/data/com.example.app/cache/
```
לפעמים אפשר למצוא כאן תמונות שהמשתמש צפה בהן, תגובות מהשרת שנשמרו, או מידע אחר שהאפליקציה שמרה לצורך ביצועים.

## אחסון חיצוני - External Storage
מלבד התיקייה הפרטית, אפליקציות יכולות לשמור קבצים גם באחסון החיצוני:
```
/sdcard/Android/data/<package_name>/
```
בניגוד לתיקייה הפרטית, באנדרואיד ישן כל אפליקציה עם הרשאת storage יכלה לגשת לקבצים האלו (גם של אפליקציות אחרות). באנדרואיד חדש (מגרסה 11) יש **Scoped Storage** שמגביל את הגישה.

## ספק תוכן - Content Provider
לפעמים אפליקציות חושפות חלק מהמידע שלהן לאפליקציות אחרות באמצעות **Content Provider**.
כזכור, content provider הוא רכיב שמאפשר שיתוף מידע בין אפליקציות.

אפשר לשאול content provider ישירות מהטרמינל:
```bash
adb shell content query --uri content://com.example.app.provider/users
```
אם הcontent provider לא מוגן כראוי (exported בלי הרשאות), כל אפליקציה יכולה לגשת למידע - וזו פגיעות אבטחה. (זה מופיע בandroid manifest)

## סיכום
כשאנחנו חוקרים אפליקציה, חשוב לבדוק את כל מקומות האחסון:
1. **SharedPreferences** - מפתחות וערכים בxml, לפעמים מכילים טוקנים וסיסמאות
2. **מסדי נתונים** - SQLite עם מידע מורכב, משתמשים, הודעות, היסטוריה
3. **קבצים** - קבצי קונפיגורציה, לוגים, מידע שהורד
4. **מטמון** - קבצים זמניים שיכולים לחשוף מידע
5. **אחסון חיצוני** - קבצים שיכולים להיות נגישים לאפליקציות אחרות
6. **ספק תוכן** - Content Providers שחושפים מידע

הרבה פגיעויות אבטחה באפליקציות נובעות מאחסון לא מאובטח של מידע רגיש. מפתחים לפעמים שומרים סיסמאות בטקסט גלוי, טוקנים בלי הצפנה, או חושפים content providers בלי הגנה מתאימה.
