
### מה הוא apk
אפליקציות Android מגיעות בקובץ בפורמט **APK** (Android Package). 
קובץ apk הוא קובץ ZIP שאפשר לפתוח כדי לראות את התוכן שלו (ממש עם winrar, או 7z)
- התוכן של קובץ APK כולל בדרך כלל: 
    - קובץ **AndroidManifest.xml** – הקובץ xml שמתאר את מבנה האפליקציה. 
		 - הרשאות שהיא צריכה,
		 - רשימה של כל הפעילויות שלה, 
		 - איזה פעילות היא הפעילות הראשית (הפעילות הראשונה שרצה כשפותחים את האפליקציה)
		 - האם פעילות מסוימת היא מיוצאת החוצה, (יכולה לקבל intent מבחוץ) או רק לשימוש פנימי.
		 - איזה actions כל פעילות מצפה לקבל. (לא חובה)
    - תקיית **META-INF/** – מכיל חתימות דיגיטליות וקבצים שמאמתים את האפליקציה. 
		- כל אפליקציה חייבת להיות חתומה דיגיטלית על ידי היוצר שלה, שירות google play מאמת את החתימה מול היוצר, ובודק האם היוצר מוכר על ידי גוגל.
    - קובץ **classes.dex** – הקוד של האפליקציה בפורמט Dalvik/DEX (הקוד שמתבצע על המכשיר).
    - תקיית **lib/** – ספריות native (sharet object- so) לפי סוגי מעבדים (ARM, x86 וכו’) (כדי לתמוך בכל סוגי המכשירים)
    - תקיית **res/** – קבצים נוספים שהאפליקציה יכולה להשתמש בהם. 
	    - תמונות, סרטונים, שמע
	    - קבצי layout: קבצי xml-ים/html-ים שמתארים מבנה של Composableים ופעילויות.
	- תקיית **assets/** - קבצים שונים, למשל פונטים.

### קוד Dalvik & Smali
- אפליקציות Android בדרך כלל נכתבות ב-Java או Kotlin, אבל פיתוח אנדרואיד הקוד המקורי לא מקומפל לjava bytecode ורץ במכונה הוירטואלית של Java (הjvm) כמו בתוכנות java סנדרטיות.
- הקוד java מקומפל ל**בייטקוד של Dalvik/DEX**.
- פורמט Smali הוא פורמט קריא-אדם של ה-bytecode הזה (בדומה ל־assembly). הכלים baksmali/smali משמשים לפיענוח והרכבה של הקוד הזה- כלומר תוכלי לכתוב קוד smali ולקמפל אותו לdex באמצעות הכלי smali, ותוכלי לרברס dex לsmali באמצעות baksmali.
- כמו שjava bytecode רץ ב״jvm" או java virtual machine ביחד עם כל הספריות של java, הdex bytecode רץ בתוך ״art" או android runtime ביחד עם כל הספריות של android (נקרא גם android sdk)

## הapktool
הכלי apktool מאפשר לנו לחקור apk-ים.
- הוא יודע לעשות unzip לכל הקבצים והתיקיות של apk
- לעשות decode לכל הmanifest 
- לרברס את הclasses.dex בחזרה smali
- גם לסדר את כל הקבצים בapk שהוא לא מכיר לתקיית ״unknown״.

## הmanifest
בשורה הראשונה של הmanifest נוכל לקבל נתונים כללים על האפליקציה.
```xml
<manifest xmlns:android="http://schemas.android.com/apk/res/android" android:compileSdkVersion="35" android:compileSdkVersionCodename="15" android:installLocation="auto" package="com.uptodown" platformBuildVersionCode="35" platformBuildVersionName="15">
```
למשל נוכל לראות את שם החבילה של האפליקציה (במקרה הזה זה com.uptodown) כי משם הורדנו את טוויטר (זה לא השם האמיתי של טוייטר).
נוכל גם לראות איזה גרסת android sdk קומפל הקוד java והגרסת אנדרואיד שאליה מיועדת החבילה (במקרה הזה זה 15)

בהמשך נוכל לראות את כל ההרשאות והפיצ׳רים שהאפליקציה מבקשת וגם באיזה סוג מסכים היא תומכת.

למעשה בדיוק כמו שיש activities (פעילויות).

- השירות- service, הוא רכיב שרץ ברקע בלי ממשק משתמש כמו פעילות. משומש לדוגמה לניגון מוזיקה ברקע, חיבור לשרת, ריצה של לוגיקה כהמשתמש יוצא ממסך.
- הrecevier הוא רכיב שמאזין לbroadcast-ים- באנדרואיד המערכת הפעלה שולחת הודעות broadcast לכל האפליקציות מדי פעם, למשל מצב הסוללה, הודעת sms שהתקבלה, שינוי חיבור רשת- והrecevier הוא קוד שאנחנו יכולים להגדיר שמקבל את המידע הזה ועושה איתו משהו.
- הcontent provider- מאפשר שיתוף נתונים בין אפליקציות, כאן אפשר להגדיר תוכן שהאפליקציה שלנו מייצאת לכל האפליקציות.
