# binfmt - איך לינוקס יודע להריץ קבצים

בהסבר הקצר הזה נבין מה זה **binfmt**, איך מערכת ההפעלה מזהה סוגי קבצים בינאריים, ואיך אפשר להוסיף תמיכה בסוגי קבצים חדשים.
## מה זה binfmt

מנגנון `binfmt` = **Binary Format**

זהו מנגנון בלינוקס שמאפשר לקרנל להבין **איך להריץ קובץ בינארי**. (קובץ הרצה)  
כאשר מריצים קובץ:

```bash
./program
```

הקרנל לא "מנחש"- הוא בודק את פורמט הקובץ לפי **binfmt handler** מתאים.
## דוגמה- ELF

רוב התוכנות בלינוקס הן בפורמט **ELF**, הפורמט המקובל לקבצי הרצה (בינארים)
כשהקרנל רואה קובץ ELF:

1. קורא את ה4 בתים הראשונים של הקובץ,וקורא את הדפוס הבא magic bytes (`0x7F ELF`)
2. משתמש ב־ELF loader (הקוד הקרנלי שיודע להריץ קבצי elf)
3. טוען לזיכרון.
4. מריץ.

למעשה, כאשר אנחנו מריצים execve על נתיב של קובץ מסוים, הקרנל מפעיל את מנגנון binfmt, שקורא את תחילת הקובץ, ויודע לשייך את הקובץ למנגנון שיודע לטעון אותו לזיכרון

## binfmt_misc -הוספת פורמטים חדשים

לינוקס מאפשר להוסיף **handlers מותאמים אישית** דרך:

```
/proc/sys/fs/binfmt_misc
```

זה נקרא:

```
binfmt_misc
```

באמצעותו אפשר לגרום ללינוקס להריץ:

- קבצי Python בלי `python file.py`
- קבצי Java
- קבצי Windows דרך Wine
- קבצי ARM דרך QEMU
- כל פורמט מותאם אישית

## איך זה עובד

כאשר מוסיפים handler, מגדירים:
- הMagic bytes או extension
- איזה interpreter להריץ
- איך להעביר לו את הקובץ
    

דוגמה רעיונית:

```
אם קובץ מתחיל ב־XYZ → הרץ /usr/bin/my_loader
```

---

## דוגמה אמיתית — הרצת קבצי ARM

מערכות רבות משתמשות בזה כדי להריץ קוד ARM על x86 בעזרת QEMU.

הקרנל מזהה ELF של ARM → מעביר ל־QEMU → QEMU מריץ.

הכל שקוף למשתמש.

## shebang מול binfmt

Shebang:

```bash
#!/usr/bin/python3
```

זה מנגנון **ברמת המשתמש** (user space).

binfmt:

- מנגנון **ברמת הקרנל**
    
- עובד גם בלי shebang
    
- מזהה לפי magic bytes

## הloader
הloader הוא רכב קרנלי שיודע לטעון קבצי הרצה לזיכרון של תהליך.
פורמט elf הוא הפורמט הלינוקסי לקבצי הרצה (בווינדוס זה pe), נצלול בקרוב לפורמט וכיצד הloader יודע לפרסר אותו.
לפני שנצלול לelf-ים, חשוב שתכירו שexecve יודע להריץ גם קבצים שהם לא בהכרח elf-ים, למשל סקריפטים bash או python, באמצעות פורמט `!#`- פורמט שכנראה ראיתם קודם.
לדוגמה:
```python
#!/bin/python

print("Hello World")
```
אם תנסו להריץ את הקובץ הבא בלינוקס עם execve, הוא יצליח לרוץ.
אם תשימו לב, בתחילת הקובץ יש #! ומיד אחרכך את הנתיב של python (קובץ הרצה- elf)
מאחורי הקלעים, הloader יריץ את הelf של python שנמצא בנתיב ויתן לו argv את הscript.
בדיוק ככה נוכל גם להריץ bash.

```bash
#!/bin/bash

echo 'Hello World'
```
וכל סוג של בינארי אחר שיודע לקבל סקריפט כפרמטר ולהריץ.

הפורמט הזה של #! נקרא shebang 
הloader יודע להתייחס לקובץ הרצה מסוים כshebang על פי ה#! שיש בהתחלה.
