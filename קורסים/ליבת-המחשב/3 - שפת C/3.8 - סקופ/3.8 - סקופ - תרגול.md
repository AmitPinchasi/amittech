

### תרגול חלק א' – משתנים ו-Scope

1. כתבו פונקציה בשם `void test()` שבתוכה מוגדר משתנה מקומי `int x = 10`, והדפיסו אותו עם `printf`. נסו לגשת אליו מ־`main()` וראו שהקומפיילר מתלונן.

2. הגדירו משתנה גלובלי `int counter = 0;` ובנו שתי פונקציות:

- `void inc()` שמעלה את `counter`

- `void show()` שמדפיסה את `counter`  
קראו להן מתוך `main()` מספר פעמים ובדקו אם הערך נשמר.

1. בתוך פונקציה, כתבו `static int counter = 0;` ואז כל פעם שהפונקציה נקראת – הגדילו את `counter` והדפיסו אותו. הריצו את הפונקציה שלוש פעמים מתוך `main()` וראו שהערך נשמר.

2. הגדירו משתנה גלובלי עם `static`, ואז נסו להצהיר עליו בקובץ `.c` אחר עם `extern`. נסו לקמפל – תראו שהוא לא נגיש. שנו ל־`int x = 5;` (בלי static) וראו שהבעיה נפתרת.

3. צרו שני קבצים:

- `a.c`: מגדיר `int value = 123;`
- `b.c`: מכיל `extern int value;` ומדפיס אותו בפונקציה `print()`
- קמפלו והפעילו את הפונקציה מתוך `main()`.

1. נסו להחליף את התחביר של האסמבלי ל־Intel syntax ע"י הוספת `asm(".intel_syntax noprefix\n ...")`
2. חפשו בגוגל על תחבירים שונים והסבירו את ההבדלים בניהם


### תרגול חלק ג' – קימפול פרויקט עם C ואסמבלי

1. כתבו קובץ אסמבלי פשוט `tools.asm` עם פונקציה `print_hello:` שכוללת רק `ret` (כרגע בלי תוכן).
2. כתבו קובץ `main.c` שמכריז על `extern void print_hello();` וקורא לה ב־`main()`.

3. קמפלו את הקבצים:

- אסמבלי:
```bash
nasm -f elf32 tools.asm -o tools.o
```

- C:
```bash
gcc -m32 -c main.c -o main.o
```

- לינקינג:
```bash
gcc -m32 tools.o main.o -o hello_program
```

1. ודאו שהתוכנית מקמפלת. בשלב הבא – הוסיפו קוד אסמבלי אמיתי להדפסת "Hello" בעזרת `int 0x80` או פשוט עדכון משתנה ש־main יכול להדפיס.

2. נסו לחלק את ההצהרה על הפונקציה האסמבלרית לקובץ `.h`, כלול אותו ב־`main.c`, ולהשתמש ב־`static` כדי להסתיר פונקציות/משתנים מקבצים אחרים.
