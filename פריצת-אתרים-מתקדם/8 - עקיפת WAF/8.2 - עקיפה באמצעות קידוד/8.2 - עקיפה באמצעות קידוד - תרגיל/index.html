    הbypass באמצעות encoding - תרגיל¶

    הכנה¶

    עבור תרגיל זה, הקימו סביבת בדיקה מקומית עם ModSecurity CRS ברמת פרנויה 2.

    # Setting up the environment with Docker
docker run -d --name modsec-test \
    -p 8080:80 \
    -e PARANOIA=2 \
    owasp/modsecurity-crs:apache

    תרגיל 1 - bypass חוקת XSS עם encoding URL¶

    האפליקציה בכתובת http://localhost:8080/search?q= מציגה את הפרמטר q בדף ללא סינון (Reflected XSS). אולם, ModSecurity חוסם payloads שמכילים <script>.

    משימה:
    1. ודאו שהpayload <script>alert(1)</script> נחסם
    2. מצאו שיטת encoding URL שעוקפת את החסימה
    3. נסו encoding URL כפול - האם עובד? מדוע כן או לא?
    4. נסו encoding חלקי - קדדו רק תו אחד מהמילה script

    תרגיל 2 - שרשרת encoding לbypass SQLi¶

    נקודת injection בכתובת http://localhost:8080/user?id=1. ה-WAF חוסם UNION SELECT.

    משימה:
    1. נסו את הpayload 1' UNION SELECT username, password FROM users-- וודאו שנחסם
    2. השתמשו בפונקציית CHAR() כדי לקודד מחרוזות
    3. השתמשו ב-hex encoding (0x) למחרוזות
    4. שלבו encoding URL כפול עם הטכניקות מסעיף 2 ו-3
    5. מצאו שרשרת encoding מינימלית שעוקפת את ה-WAF

    תרגיל 3 - ישויות HTML לbypass XSS¶

    נקודת injection בתוך attribute של HTML:

    <input type="text" value="USER_INPUT">

    ה-WAF חוסם onerror, onload, onfocus ו-alert.

    משימה:
    1. בנו payload שיוצא מה-attribute ומריץ JavaScript
    2. השתמשו בישויות HTML עשרוניות (decimal entities) לencoding event handler
    3. השתמשו בישויות HTML הקסדצימליות עם אפסים מובילים
    4. שלבו ישויות HTML עם Unicode escapes בתוך ה-JavaScript
    5. מצאו payload שעובד עם לפחות שתי שכבות encoding

    תרגיל 4 - encoding Unicode לbypass¶

    ה-WAF חוסם את המילים: alert, prompt, confirm, eval, script.

    משימה:
    1. כתבו payload XSS שמשתמש ב-Unicode escape sequences (\u00XX) כדי לעקוף
    2. כתבו payload שמשתמש בתווי Unicode ברוחב מלא (full-width)
    3. כתבו payload שמשתמש ב-hex escapes (\xXX)
    4. כתבו payload שמשלב Unicode escapes עם אוקטלי
    5. בדקו: האם הpayloads עובדים אם ה-WAF מפעיל t:utf8toUnicode בחוקים?

    תרגיל 5 - Base64 ו-JSFuck¶

    משימה:
    1. כתבו payload XSS שמשתמש ב-atob() לdecoding Base64 בזמן ריצה:
    - קדדו alert(document.cookie) ל-Base64
    - בנו payload שמשתמש ב-eval(atob(...))
    - עטפו את הכל בתגית HTML מתאימה
    2. כתבו את הפקודה alert(1) ב-JSFuck (ניתן להשתמש בממיר אונליין)
    3. מצאו הקשר שבו payload JSFuck יעבוד בפועל (איזה תגית HTML תתמוך?)

    תרגיל 6 - Hackvertor ב-Burp Suite¶

    משימה:
    1. התקינו את התוסף Hackvertor ב-Burp Suite
    2. בנו שרשראות encoding הבאות ב-Hackvertor ובדקו אותן:
    - הencoding URL כפול של payload XSS
    - ישויות HTML בתוך encoding URL
    - Base64 בתוך encoding URL
    3. צרו שרשרת encoding מותאמת אישית שעוקפת את כל החוקים הבאים בו-זמנית:
    - חסימת <script>
    - חסימת alert
    - חסימת onerror

    תרגיל 7 - אתגר מסכם¶

    בקשה לשרת הבדיקה:

    POST /api/comment HTTP/1.1
Host: localhost:8080
Content-Type: application/x-www-form-urlencoded

comment=USER_INPUT

    ה-WAF (ModSecurity CRS ברמת פרנויה 3) חוסם:
    - כל תגיות HTML
    - כל event handlers
    - את המילים alert, prompt, confirm, eval, Function
    - ישויות HTML
    - javascript: protocol

    משימה:
    1. מצאו דרך להריץ JavaScript למרות כל החסימות
    2. תעדו כל ניסיון - מה נחסם ומדוע
    3. הגיעו לpayload עובד עם מינימום שכבות encoding
    4. כתבו סקריפט Python שמייצר את הpayload אוטומטית ושולח אותו
