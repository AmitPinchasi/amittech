    0.2 - הפרויקט

    הפרויקט: TaskFlow¶

    לאורך הקורס נבנה מערכת לניהול משימות בשם TaskFlow, בסגנון Trello או Jira מפושט.

    מה המערכת עושה?¶

    • משתמשים יכולים להירשם ולהתחבר
    • כל משתמש יכול ליצור פרויקטים
    • בכל פרויקט אפשר ליצור משימות עם כותרת, תיאור ותאריך יעד
    • משימות יכולות להיות במצבים שונים: "לעשות", "בתהליך", "הושלם"
    • משתמשים יכולים להוסיף תגובות למשימות

    איך הפרויקט מתפתח?¶

    בכל שלב בקורס נחזור לפרויקט ונשפר את הארכיטקטורה שלו:

    • שלב 1 - עקרונות עיצוב: הקוד הראשוני עם בעיות ידועות - נזהה אותן ונתקן
    • שלב 2 - תבניות עיצוב: נוסיף תבניות שפותרות בעיות ספציפיות
    • שלב 3 - ארכיטקטורות מערכת: נחליט איך לחלק את המערכת לשכבות ושירותים
    • שלב 4 - עיצוב בקנה-מידה: נתכנן איך TaskFlow יראה עם מיליון משתמשים

    נקודת ההתחלה¶

    הנה הקוד הראשוני. קראו אותו ונסו לזהות בעצמכם את הבעיות לפני שממשיכים לפרק הבא.

    # main.py - everything in one place
from fastapi import FastAPI
import sqlite3
import hashlib

app = FastAPI()

def get_db():
    return sqlite3.connect("taskflow.db")

@app.post("/register")
def register(username: str, password: str):
    db = get_db()
    hashed = hashlib.md5(password.encode()).hexdigest()
    db.execute("INSERT INTO users VALUES (?, ?)", (username, hashed))
    db.commit()
    return {"message": "Registered successfully"}

@app.post("/tasks")
def create_task(title: str, description: str, user_id: int):
    db = get_db()
    db.execute("INSERT INTO tasks VALUES (?, ?, ?)", (title, description, user_id))
    db.commit()
    return {"message": "Task created"}

@app.get("/tasks")
def get_tasks(user_id: int):
    db = get_db()
    tasks = db.execute(
        "SELECT * FROM tasks WHERE user_id = ?", (user_id,)
    ).fetchall()
    return tasks

@app.put("/tasks/{task_id}")
def update_task(task_id: int, title: str, description: str):
    db = get_db()
    db.execute(
        "UPDATE tasks SET title=?, description=? WHERE id=?",
        (title, description, task_id)
    )
    db.commit()
    return {"message": "Task updated"}

@app.delete("/tasks/{task_id}")
def delete_task(task_id: int):
    db = get_db()
    db.execute("DELETE FROM tasks WHERE id=?", (task_id,))
    db.commit()
    return {"message": "Task deleted"}

@app.post("/tasks/{task_id}/comments")
def add_comment(task_id: int, user_id: int, content: str):
    db = get_db()
    db.execute(
        "INSERT INTO comments VALUES (?, ?, ?)", (task_id, user_id, content)
    )
    db.commit()
    return {"message": "Comment added"}

@app.post("/tasks/{task_id}/notify")
def notify_assignee(task_id: int):
    db = get_db()
    task = db.execute(
        "SELECT * FROM tasks WHERE id=?", (task_id,)
    ).fetchone()
    user = db.execute(
        "SELECT email FROM users WHERE id=?", (task[2],)
    ).fetchone()
    # Sending email directly from the route
    import smtplib
    server = smtplib.SMTP("smtp.gmail.com", 587)
    server.sendmail("system@taskflow.com", user[0], f"Task {task[0]} updated")
    return {"message": "Notification sent"}

    מה הבעיות בקוד?¶

    חשבו על השאלות הבאות:

    • מה קורה אם רוצים לשנות את מסד הנתונים מ-SQLite ל-PostgreSQL?
    • מה קורה אם רוצים לשנות את שיטת ההצפנה מ-MD5 למשהו יותר בטוח?
    • מה קורה אם רוצים להוסיף בדיקות (tests) לקוד?
    • מה קורה אם צריך לשלוח התראות גם ב-SMS וגם במייל?
    • מה קורה אם הקוד גדל ל-50 routes - איך מחלקים אותו?

    בפרקים הבאים נענה על כל השאלות האלו.
